# MD 讲义格式规范 · 原子结构样板

> **一句话**：这是 MD→LaTeX→PDF 管线的目标输入规范。新讲义、重构稿、预检器规则，都以本文档为准。
> **定位**：本文档只定义 Markdown 规范；编译命令、脚本行为和报错处理请看 [[11-模板/LaTeX PDF管线使用手册]]。

## 适用范围与当前边界

- 本文档定义的是**目标规范**，优先约束之后的新讲义与重构稿。
- 当前样板 [[原子结构-超级充实版（自学完整）.md]] 已完成“规范化样板 v1”清洗，可作为结构参考。
- 现存少量历史兼容写法如果已经手工修通，可以暂不追改；但**不要继续复制到新稿里**。
- 当前图片策略分两层：
  - **来源仓**：`mineru/`、教材 OCR、真题原图、真题解析等，用来找图。
  - **最终落地图**：vault 根目录 `media/`，用来给讲义和 PDF 管线稳定引用。

## 核心原则（30 秒速查）

| 元素 | 正确写法 | 不再推荐 |
|:---|:---|:---|
| 图片引用 | `![[media/xxx.png]]` | `![[mineru/...]]` 直接进讲义 |
| 图注 | `*图 N 对象 + 用途*` | `<center>` / 长说明 / 来源尾注 |
| callout | `> **理解要点**：...` | 旧 emoji callout 混用 |
| 难度标记 | `基础 / 进阶 / 综合 / 挑战` | 旧星级难度写法 |
| 简单下标 | `mₛ`、`mₗ`、`dₓᵧ` | 把所有短记号都硬塞进数学模式 |
| 复杂表达式 | `$d_{xz}$`、`$d_{z^2}$`、`$E_{4s}<E_{3d}$` | 用不完整 Unicode 凑写 |
| e_g / t_2g | `$e_g$`、`$t_{2g}$`（必须用 LaTeX 下标） | `e_g` 裸下划线格式 🔥 |
| Unicode 箭头 | `→` `↑` `↓`（直接写） | `$\rightarrow$`（CJK 字间距异常）🔥 |
| 罗马数字 | `Ⅱ` `Ⅲ` `Ⅳ`（直接写 Unicode） | 手动转 ASCII |
| 来源说明 | 放正文，不放图注 | `*来源：某教材*` |

## 一、Frontmatter（YAML）

```yaml
---
title: 学生讲义-讲义名（超级充实版·自学完整）
type: 学生讲义
source_book: "..."
chapter: 第X讲
serve_rounds: [第一轮, 第二轮]
serve_topics: [标签1, 标签2]
difficulty_level: 基础~挑战
related_notes:
  - "[[04-课件/备课大纲/xxxx]]"
  - "[[专题-xxx]]"
tags: [学生讲义, 超级充实版, xxx]
created: YYYY-MM-DD
has_images: true
image_count: N
updated: YYYY-MM-DD
last_audit: "..."
template_version: 自学完整版 vX.X
exemplar: true
---
```

**规则**

- `difficulty_level` 使用中文难度区间，不再使用 `⭐`。
- `image_count` 必须与正文中的 `![[media/...]]` 数量一致。
- `related_notes` 优先挂对应备课大纲、专题页、核心 KP。
- `template_version` 必须带版本号，便于预检器识别样板代际。

## 二、标题下元信息块

标题下方统一用引用块承载元信息：

```markdown
# 第X讲：标题

> **对应专题**：[[专题-xxx]]
> **对应备课大纲**：[[04-课件/备课大纲/xxxx]]
> **建议使用方式**：课前→课堂→课后
> **前置要求**：......
> **深度边界**：......
> **课时**：第X课（日期）
> **版本说明**：本文件为自学完整版。
> **记号约定**：简单下标用 Unicode，复杂表达式用数学模式。
```

**规则**

- 元信息块只放文件级说明，不放知识性正文。
- 行首统一 `> **字段**：`。
- 这部分属于“顶部元信息”，供管线识别并在 PDF 前处理时剥离。

## 四、正文与强调

### 1. 普通段落

- 直接书写，避免无意义空行堆积。
- 概念名、关键词、结论可用 `**加粗**`。

### 2. 列表

```markdown

```

### 3. 分隔线

- 使用 `---`
- 不再使用中文长破折号作为结构分隔符

## 五、数学与化学式

### 1. 行内数学

```markdown
$E = mc^2$
$\psi$ 的 $\|\psi\|^2$ 为概率密度
$\tilde{\nu} = R_H \left(\frac{1}{2^2} - \frac{1}{n^2}\right)$
```

### 2. 显示数学

```markdown
$$
\boxed{E_n = -\frac{13.6}{n^2}\ \mathrm{eV}}
$$
```

### 3. 化学式

```markdown
$\ce{^{238}_{92}U -> ^{234}_{90}Th + ^{4}_{2}He^{2+}}$
```

### 4. 下标边界

**可以直接用 Unicode 的情况**

- `mₛ`
- `mₗ`
- `dₓᵧ`

**必须回到数学模式的情况**

- 出现 `z`
- 出现平方、负号、上下标组合
- 出现比较式、能量式、量子数组

```markdown
$d_{xz}$
$d_{yz}$
$d_{z^2}$
$d_{x^2-y^2}$
$E_{4s}<E_{3d}$
$(n,l,m_l,m_s)$
```

**结论**

- 简单短记号用 Unicode，保证阅读自然。
- 复杂表达式用数学模式，保证 PDF 稳定。

### 5. 化学轨道符号（e_g / t_2g）🔥

> **铁律**：`e_g`、`t_2g` 等轨道符号在正文中**永远用 `$e_g$` LaTeX 下标**，不要用 `_` 纯文本格式。

| 场景 | 正确写法 | 错误写法 |
|:---|:---|:---|
| 正文/图注 | `$e_g$ 轨道` | `e_g 轨道`（`_` 不渲染为下标） |
| 已在 `$...$` 中 | `$t_{2g}^3 e_g^1$` ✅ | — |
| 表格单元格 | `$e_g$` 或 `$t_{2g}$` | `e_g`（裸下划线） |
| `n(t₂g)` 写法 | `n($t_{2g}$)` | `n(t₂g)`（`₂` 可能不渲染） |

**原因**：preprint 函数会将正文中的 `e_g` → `$e_g$`，但若源文件中 `e_g` 出现在 `$...$` 内部（如 `_{e_g}`），会导致嵌套 `$...$`，PDF 编译报错。

### 6. Unicode 符号与箭头 🔥

> **铁律**：`→` `↑` `↓` `≈` `≤` `≥` `≠` `σ` `π` 等 Unicode 符号**直接写在正文中**，preprint 会通过 preamble 宏自动渲染，**不要手动替换为 LaTeX 命令**。

| 符号 | 正确写法 | 错误写法 |
|:---|:---|:---|
| 箭头 | `→`（直接写） | `$\rightarrow$`（会导致 CJK 字间距异常） |
| 约等号 | `≈`（直接写） | `$\approx$`（同上） |
| 希腊字母 | `σ` `π`（直接写） | `$\sigma$`（正文中间插入 `$...$` 切断词边界） |
| 热力学 ° | `ΔᵣH°`（直接写） | `$\Delta_{r}H^{\circ}$`（preprint 自动处理） |

**原因**：XeLaTeX 中 `$...$` 被视为词边界，在 CJK 字符之间插入会导致字间距异常（每个字之间出现空格）。preamble 中的 `\newcommand{\箭}{\ensuremath{\rightarrow}}` 宏通过 `\ensuremath` 避免此问题。

### 7. 罗马数字

> 配合物价态等罗马数字**直接写 Unicode**（`Ⅰ` `Ⅱ` `Ⅲ` `Ⅳ` `Ⅴ` `Ⅵ` `Ⅶ` `Ⅷ`），preprint 会自动转为 ASCII。

| 场景 | 写法 |
|:---|:---|
| 配合物价态 | `钴(Ⅲ)` → preprint → `钴(III)` |
| 族标签 | `ⅡA` `ⅤA` → preprint → `IIA` `VA` |

### 8. H1 标题命名

> H1 标题**不要包含**"超级充实版""自学完整"等后缀，否则会出现在 PDF 页眉中。

```markdown
# 第X讲：讲义名称          ← 正确
# 第X讲：讲义名称-超级充实版（自学完整）  ← 错误（会进页眉）
```

## 七、图片

### 1. 最终引用格式

```markdown
![[media/atomic-structure-h-spectrum.png]]
![[media/atomic-structure-h-spectrum.png|420]]

*图 1 氢原子光谱与跃迁对应关系*
```

### 2. 路径规则

- 新讲义最终图片统一放在 vault 根目录 `media/`
- 正文统一写 `![[media/文件名.ext]]` 或 `![[media/文件名.ext|宽度]]`
- 不在最终讲义里直接保留 `![[mineru/...]]`
- 不使用 `![[media/*.md]]`
- 不再保留多套 `media/` 路径；所有正式图片只留在根目录 `media/`

### 3. 文件命名规则

- 使用稳定英文名，推荐 `topic-keyword-purpose.ext`
- 文件名应直接表达内容对象，不用 `image1-final-final` 这类模糊名
- 不再新增 `Pasted image ...`
- 不新增中文临时名、空格名、同图多副本名
- 同一张图不为了撑版面做相邻重复摆放

### 4. 图注规则

- 图注格式统一为 `*图 N 对象 + 用途*`
- 图注控制在 30 汉字左右
- 不写来源说明
- 详细解释放回正文，不塞进图注
- 同一图如果需要补充限制条件，优先回正文写成 `> **读图提示**：...`

### 5. 格式建议

- 默认优先 `.png`
- 扫描图可用 `.jpg` / `.jpeg`
- `.svg` 仅在本地编译验证通过时使用，否则先转 `.png`

### 6. 使用约束

- 新稿正文只保留最终落地图，不混用找图过程中的中间路径
- 同一知识点如需多图，优先“不同图各司其职”，不要用重复图并排充版
- 一张图只承担一个主要教学功能：定义、结构、对比、读图推理，尽量不要一图塞满

### 7. 找图入口

- 找图先看 [[00-首页/图片索引]]
- 来源选择规则看 [[00-首页/规则-配图来源优先级]]

### 8. 当前样板的例外

若某个旧引用已经手工修通且不值得再扰动，可以保留为历史兼容；  
但它**不是**新稿应继续复制的模板。

## 八、教学提示块（统一 callout 风格）

统一使用一行式 blockquote：

```markdown
> **理解要点**：......
> **易错提醒**：......
> **核心辨析**：......
> **名师点拨**：......
> **理解方法论**：......
> **假设一**：......
```

**规则**

- 保留 `> **标签**：内容` 这一套，不再混用 emoji callout。
- 标签文字尽量稳定、可枚举，方便后续预检器识别。
- 一条 callout 尽量只表达一个教学功能。

**推荐标签**

- `理解要点`
- `易错提醒`
- `核心辨析`
- `名师点拨`
- `理解方法论`
- `读图提示`
- `假设一 / 假设二 / 假设三`

## 十一、当前管线会做什么

`convert_handout_to_pdf.py` 的 `preprint()` 当前会做这些事：

| 输入 | 处理 |
|:---|:---|
| YAML frontmatter | 删除 |
| 顶部元信息行 `> **对应专题**：...` 等 | 删除 |
| 教学 callout `> **理解要点**：...` 等 | 保留 |
| `![[media/xxx.png]]` | 转成 Pandoc 图片语法 |
| `[[wikilink]]` | 剥离为纯文本 |
| `⭐` / `★` | 编译时会清掉，但源文件不应继续使用 |
| 旧 `<center>图 ...</center>` | 兼容转换为图注 |

**重要边界**

- “能编译”不等于“是好规范”。
- 预检器与新稿，应按本文档校验，而不是依赖历史兼容。

- [ ] 图片统一写成 `![[media/...]]`

- [ ] 例题与练习不再使用 `⭐`
- [ ] `image_count` 与正文图片数一致

## 十三、给 PDF 预检器的重点规则

后续做预检器时，优先检查这些高价值问题：

1. 是否仍有 `⭐` / `★`
2. 是否仍有 `![[mineru/...]]` 直接进入学生讲义
3. 是否仍有 `Pasted image ...`
4. 图注是否过长
5. 是否存在相邻重复图或明显重复占位图
6. 是否混用 emoji callout 与 `> **标签**：...`
7. 是否把应当用数学模式的复杂下标写成残缺 Unicode
8. `image_count` 是否与正文一致

这 8 条会直接决定讲义是否容易继续维护。
