# 图片替换决策树

> **用途**：当讲义中某张图片需要替换时（图注对不上、信息不准确、风格不匹配、清晰度不够），提供标准化的决策路径，避免"在本地相似图里反复试"的低效循环。
> **核心原则**：**明确目标 → 查已有 → 查来源仓 → 直接走网络 → 不在相似图里反复试**
> **提炼来源**：2026-06-30 分子结构基础 图 2 "离子极化效应"替换实战（3 次本地失败 → 网络资源 1 次通过）

---

## 一、什么时候需要图片替换

| 触发场景 | 示例 |
|:---|:---|
| 图注与图片内容不匹配 | 图注写"离子极化效应"但图片是简单的离子键示意图 |
| 图片信息量不足 | 示意图太简单，无法支撑教学说明 |
| 图片风格与讲义不一致 | 其他图都是高质量教材级，某张是低分辨率手绘 |
| 图片重复放置 | 同一张图在讲义中出现了两次 |
| 知识点升级后原图不再适用 | 讲义重组后，某张图的位置和功能发生了变化 |

---

## 二、决策树（核心流程）

```
需要替换图片
  │
  ▼
┌─────────────────────────────────────┐
│ Q1: 这张图需要什么？                │
│ （明确替换目标：视角/信息/风格）      │
└─────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────┐
│ Q2: [[图片索引]] 中有没有可复用的？  │
│ 搜索关键词：知识点名 + 用途          │
└─────────────────────────────────────┘
  │
  ├── 有 → 验证视觉效果 → 合适则采用 → 落地 media/
  │
  ▼ 无
┌─────────────────────────────────────┐
│ Q3: mineru/ 来源仓有高清原图吗？     │
│ 注意：仅当需要"同教材风格"时走此步    │
└─────────────────────────────────────┘
  │
  ├── 有 → Read 验证（防 OCR 噪声） → 合适则复制到 media/
  │
  ▼ 无 或 风格不匹配
┌─────────────────────────────────────┐
│ Q4: 直接走网络资源                   │
│ （⚠️ 不要再回头试本地相似图！）       │
└─────────────────────────────────────┘
  │
  ▼
选择来源 → 下载 → 验证 → 落地 media/ → 更新讲义
```

### ⛔ 关键禁令

> **不要在本地相似图中反复尝试超过 1 次。**
> 如果第 1 次从 mineru/ 找到的图不合适，第 2 次仍然不合适的概率 >80%（同一教材的绘图风格高度统一）。此时应直接跳到网络资源。

## 三、网络资源来源优先级

| 优先级 | 来源 | 优势 | 适用场景 |
|:---:|:---|:---|:---|
| **1** | **Wikimedia Commons** | CC 协议、稳定 URL、高质量科学图 | 概念示意图、电子云分布、分子结构图 |
| **2** | **教材配套网站** | 与教学内容高度匹配 | 教材级图表、实验装置图 |
| **3** | **学术图库**（如 PubChem、PDB） | 专业数据可视化 | 分子结构、光谱数据、晶体结构 |
| **4** | **其他开放资源** | 覆盖面广 | 特殊视角、历史图片、工业应用 |

### 3.1 Wikimedia Commons 搜索技巧

```
搜索关键词组合：
- 英文概念名 + "diagram" / "schematic" / "illustration"
- 例如："covalent ionic bond electron cloud"
- 例如："electronegativity periodic trend"
- 例如："sigma pi bond orbital overlap"

筛选条件：
- License: CC BY / CC BY-SA / CC0
- 分辨率: ≥ 800px 宽
- 格式: SVG > PNG > JPG
```

### 3.2 下载与落地规范

1. **下载**：使用 curl/wget 下载到 vault 根目录 `media/`
2. **命名**：使用描述性英文名，格式 `topic-keyword-purpose.ext`
   - ✅ `covalent-ionic-bonds-wikipedia.jpg`
   - ❌ `image1-final-final.png`、`Pasted image ...`
3. **验证**：Read 打开图片确认内容正确
4. **讲义更新**：
   - 替换 `![[media/旧图.jpg]]` 为 `![[media/新图.jpg|宽度]]`
   - 更新图注（如有变化）
   - 更新 frontmatter `image_count`
5. **来源注释**：如果图片来自网络，在讲义正文中该图之前加一行来源说明：
   ```markdown
   > 图片来源：Wikimedia Commons, CC BY-SA 4.0
   ```

## 四、替换后必须做的事

| 步骤 | 操作 | 工具 |
|:---|:---|:---|
| 1. 视觉验证 | Read 打开新图片，确认内容与图注匹配 | Read |
| 2. 图注更新 | 确认图注准确描述新图片内容 | 手动 |
| 3. image_count 同步 | 更新 frontmatter 中的 image_count | Edit |
| 4. 图片唯一性检查 | 确认没有同图重复放置 | `grep + uniq -d` |
| 5. 预检 | 跑 `pdf_preflight.py` 确认 0 error | PowerShell |

## 五、经验教训（来自实战）

### 5-1 本地相似图的陷阱 🔥

> **教训**：同一教材的 OCR 提取图片风格高度统一（同一作者的绘图），在需要"完全不同视角"的替换场景下几乎没有区分度。
>
> **正确做法**：明确目标后，最多试 1 次本地来源；不满意立即转网络资源。

### 5-2 图注与图片内容不匹配

> **教训**：图注写的是"离子极化效应"，但本地找到的图片都是简单的离子键示意图，无法展示"极化→电子云变形"的物理图像。
>
> **正确做法**：先明确"这张图需要传达什么信息"，再去找图。信息需求决定图的选择，不是反过来。

### 5-3 网络图片的协议与稳定性

> **教训**：从普通网站直接截图的图片可能有版权问题，且 URL 不稳定。
>
> **正确做法**：优先 Wikimedia Commons（CC 协议 + 永久 URL）。下载后存入 `media/` 作为 vault 资产，不再依赖外部 URL。
