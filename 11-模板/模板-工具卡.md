# 模板：工具卡

> 使用方式：本模板用于生成“单页可调用”的课堂工具卡，落到 `04-课件/工具卡/`。
>
> 职责定位：不是替代完整讲义，而是把某个专题里最常调用的判断按钮压缩成一页。
>
> 推荐场景：课前 2 分钟回忆、课中板书骨架、课后第一轮复习。

---

## 一、适用标准

- 适合做成工具卡的内容：
  - 有明确的第一判断按钮
  - 能压成 3-5 个分流节点
  - 常在不同专题间重复调用
- 不适合直接做成工具卡的内容：
  - 需要完整推导才能成立的理论段落
  - 主要靠题感和长例题展开的内容
  - 一页内无法压清楚的复杂系统

## 三、写作原则

1. 先写“按钮”，再写知识，不要把讲义缩写一遍。
2. 每一行都要服务于“快速判断”，不要堆背景解释。
3. 优先写学生和老师都能现场调用的话术。
4. 一张卡最好只解决一个核心判断问题，必要时拆卡。

---

## 四、推荐长度

| 区块 | 建议长度 |
|:---|:---|
| 一句话定场 | 3 行以内 |
| 核心分流表 | 4-8 行 |
| 对照表 | 3-6 行 |
| 板书极简版 | 3-5 个按钮 |
| 使用提醒 | 2-4 条 |

---

## 五、和其他材料的分工

| 材料 | 解决什么问题 |
|:---|:---|
| 完整讲义 | 把知识讲明白 |
| 备课大纲 | 把课怎么讲设计清楚 |
| 总卡/总索引 | 先找到大赛道 |
| 单页工具卡 | 把局部判断压成秒用工具 |

---

## 六、质量检查清单

- [ ] 第一判断按钮是否足够明确
- [ ] 是否能在 10-20 秒内完成首轮分流
- [ ] 是否避免把完整讲义原封不动压缩进来
- [ ] 是否已经写明关联专题和回链入口
- [ ] 是否标明来源提炼，便于后续回源补料
- [ ] 是否适合课中直接板书或投屏使用
