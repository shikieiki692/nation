| 网课中的高价值表达已吸收 | 是 / 否 | 写对应讲法或提醒 |
| KP / 专题已有资产已复用 | 是 / 否 | 写复用页面 |
| 至少有 1 组例题链 | 是 / 否 | 写题目来源 |
| 至少有 1 组易错点 | 是 / 否 | 写易错主题 |
| 至少有 1 处图片或图景支持理解 | 是 / 否 | 写图片来源 |
| 至少有 1~2 处机制 / 推导支撑 | 是 / 否 | 写对应板块 |
| `referenced_kps` 大部分都有实际落点 | 是 / 否 | 写“哪些 KP 分别贡献了什么” |
| 考纲外但教材核心的内容已做取舍说明 | 是 / 否 | 写保留或不保留的理由 |

## 3. 交付物清单

| 交付物 | 面向对象 | 格式 | 落点 | 完成标准 |
|:---|:---|:---|:---|:---|
| 教师上课大纲 | 教师 | `.docx` | `04-课件/试点产出/` | 结构完整，可直接打印或课堂查看 |
| 学生讲义 | 学生 | `.docx` | `06-学生侧材料/讲义/` | 版式清楚，可打印发放 |
| 投影课件 | 课堂投影 | `.pptx` | `04-课件/试点产出/` 或 `04-课件/PPT/` | 页序合理，投影可读 |

### 3.0 进入 PPT 前的准入检查

- [ ] 备课大纲的课时结构、主问题、例题顺序、媒介分工已稳定
- [ ] 学生讲义已完成“五件套”核对，不再只是骨架版
- [ ] 大纲与讲义对主链、边界、例题顺序没有打架
- [ ] 已明确哪些内容必须投影，哪些内容保留板书或讲义
- [ ] 当前做 PPT 的目的，是压课堂节奏与投影界面，而不是补内容空洞

## 4. 抽料规则

### 4.0 默认优先级

1. 教材主干与备课框架：决定“这节课讲什么”
2. KP / 专题现成资产：负责结构化复用
3. 网课资料：负责补讲法、类比、易错提醒
4. 题库 / 真题链：负责把内容坐实为例题、变式、练习
5. 图片资产：负责理解增益与投影支持

### 4.0B 内容做厚底线

1. 先把主教材稳定出现的主干内容讲全，再考虑删减与压缩
2. 不允许只保留“定义 + 一页例题”式空壳结构
3. 每个核心板块默认至少检查：
   - 定义 / 对象
   - 原理 / 推导 / 机制
   - 对照 / 分类 / 反例
   - 例题 / 变式
   - 易错点 / 纠偏
4. 若教材中已有稳定板块（如定律、图景、推导、典型物性解释、典型结构），默认应进入讲义或教师稿，不能只在加工页写“已参考”
5. 内容增厚优先靠教材、网课高价值讲法、KP、题库和图片，不靠空泛扩写
6. 不用统一页数约束不同专题；同类课型之间页数可差很多，只要“该讲的内容都讲清了”即可
7. 后续新增资料默认继续并入抽料审计与内容覆盖核对，不另起临时流程

### 4.0A 大量 KP 场景下的默认推进顺序

1. 主教学链桥梁 KP
2. 外扩链桥梁 KP
3. 高频题型 KP
4. 高频易错 KP
5. 高频图景 KP
6. 枝节 / 特殊补充 KP

> 不默认按目录顺序推进，也不默认按“谁先看到就补谁”推进。

> 考纲主要用于校边界，而不是机械删教材内容。教材稳定讲、课堂有价值、又不明显过偏的内容，可以保留，但要写清取舍理由。

### 4.1 教师稿从哪里抽

- 课时分配、媒介分工、边界决策：优先取自备课大纲
- 教学预警、例题链、课堂追问：优先取自备课大纲 / 教学洞察

### 4.2 学生讲义从哪里抽

- 学习目标、知识内容、例题、易错点、作业：优先取自新授课 / 习题课正文
- 教材上稳定出现、且在本节边界内的定律、推导、图景、概念定义，应补进讲义正文，不要只保留标题
- 网课资料中课堂表达特别清楚的类比、分层讲法、易错提醒，可转写为讲义说明或旁批
- `KP` 不是只拿来挂 frontmatter，要明确分工：
  - 主干 KP 提供定义 / 定律 / 主线结论
  - 支撑 KP 提供机制 / 推导 / 比较维度
  - 题型 KP 提供例题链 / 变式 / 真题入口
  - 易错 KP 提供误区 / 反例 / 纠偏话术
  - 图景 KP 提供图片 / 表格 / 图像说明
- 题库 / 真题链中已经验证有代表性的题目，应至少抽 1 组“例题 + 变式 / 易错对照”，不要只靠正文空讲
- 考纲没有明确列出、但主教材和备课框架稳定保留的内容，可以讲，但要在加工页中写清“保留理由”
- 每个核心板块默认过一遍“五件套”：定义 / 原理或推导 / 例题 / 易错点 / 练习
- 学生讲义的默认收口方式改为：
  - 正文把主线和关键解释讲透
  - 例题数量少而精，必须服务主链或高频误区
  - 作业极简，不把大量知识后移给课后完成
- 若某专题更偏普化主干 / 高中衔接、而非高频竞赛拉分专题，默认进一步降低作业比重，把空间让给课堂解释、对照与图景
- 不得写入教师侧边界语句

### 4.2A 学生讲义优先补强的默认顺序

1. 主问题和判断主链
2. 必需的原理 / 推导 / 机制解释
3. 例题链与变式链
4. 易错点与纠偏话术
5. 图景与教材插图
6. 最后才是作业和附录美化

> 若讲义看起来“页数不短”，但前 1~4 项仍偏薄，默认仍视为没有真正搞清楚。

### 4.3 PPT 从哪里抽

- 标题页、问题页、表格、流程图、投影题干、总结页：可从备课大纲与讲义抽取
- 长推导、现场推演、需要师生同步构建的内容：默认保留板书，不一次性塞进 PPT
- 一页只承载一个核心任务：定义 / 图景 / 公式 / 例题 / 小结 其一
- 不把讲义段落原封不动贴进 PPT；能压成表、图、步骤、短句就压
- PPT 需要的图片优先来自教材图、知识库现有图、mineru 图，不用装饰性图片凑页

### 4.3A 当前 PPT 体系的改进口径（2026-06-19）

> 基于本地试点和现有 `pptx` skill，总结当前更值得执行的改进方向：

1. `PPT` 默认只承担“投影必要内容”，不要再承担内容定稿职能
2. 文本密度高的教学页，优先改成 `表格 / 两栏 / 判断顺序卡`，减少自由流式 HTML 长段落
3. 样板专题先通过 `大纲 + 讲义` 冻结主链，再复制成 PPT 骨架，避免每门课重新设计
4. 真题、图景、课堂口令更适合在大纲/讲义层先定，再抽到 PPT，而不是反过来
5. `PPT` 验收必须带缩略图或预览检查；没有视觉复检，就不应判定为稳定终版

## 5. 媒介分工决策

| 内容 | 最佳媒介 | 处理规则 |
|:---|:---|:---|
| 结构化对比表 | PPT | 直接投影 |
| 长公式推导 | 板书 + 讲义 | PPT 只放关键节点 |
| 真题原题 | PPT | 节省抄题时间 |
| 过程性解题链 | 板书 | 让学生跟随构建 |
| 课后速查卡 | 讲义附录 / 单独 docx | 便于打印裁切 |
| 定义、定律、基本推导 | 讲义 + 板书 / 局部 PPT | 先让学生看清主线，再决定投影压缩 |

## 6. 样式与技术约束

- 统一遵循 [[11-模板/模板-课件产出规范]]
- 关键公式默认优先使用 Word 原生数学对象（OMML），不要默认走公式图片
- 短化学式与简单上下标优先用正常文本排版；只有原生数学对象明显不够时，才使用公式图片兜底
- 化学式、上下标、根号、分式需做规范检查，避免 `_` 式伪下标长期留在学生正文
- 若使用公式图片，必须检查字符间距、上下标距离、化学式数字位置是否接近教材排版，而不是“能看见就算过”
- 字体、命名、脚本路径均以该规范为准
- `PPT` 若包含英文标题角标、分子式、变量、单位或公式，必须在加工页中明确写出本轮是否已做“中文标题黑体 / 中文正文宋体（必要时黑体） / 英文数字变量 `Times New Roman`”收敛
- `PPT` 中教师提示框、页序建议、底部提醒条、窄栏侧注、卡片内两行以上小字，必须单独记录是否做过投屏检查；若宋体偏虚，先缩句、再加字号，必要时切黑体
- Windows 本地 render 若通用 `render_docx.py` 失败，优先切换到 `11-模板/scripts/render_docx_windows.py`
- Windows 本地包装器应保持自足，不要依赖插件缓存目录中的固定版本脚本路径
- 日常调用优先走 `11-模板/scripts/render-docx.ps1` 或 `render-docx.cmd`，不要每次手写长 Python 命令
- 默认输出目录建议采用同目录 `<文件名>_render/`，便于把渲染产物和正式交付物分开
- 若目标文件被占用，允许以 `-v2` 作为备用输出名

## 7. 产出顺序

1. 先冻结主真相源，不在导出阶段大改正文
2. 先出教师上课大纲 `.docx`
3. 再出学生讲义 `.docx`
4. 最后出投影课件 `.pptx`
5. 每次导出后都要 render 检查，再记录版式问题、缺图、超长页、公式异常
