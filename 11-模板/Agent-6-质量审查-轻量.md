# Agent 6 · 质量审查（轻量版）

## 一句话定位
审查已填充知识点的质量和准确性。

## 何时使用
- 一批知识点填充完成后需要验收
- 定期质量抽检
- 发现内容可疑需要复核

## 核心输入（必填）
| 参数 | 说明 |
|------|------|
| 审查范围 | 知识点文件路径列表或目录模式 |
| 审查批次 | 超过20个知识点需分2-3批，每批~10个 |

## 执行流程（3步）
1. **逐文件审查**：按6个维度评分（完整性、准确性、结构化、链接完整性、竞赛适切性、可检索性）
2. **输出报告**：每文件给出总分、具体问题、修改建议
3. **归档记录**：审查报告保存到 08-待审核区/

## 完成定义（DoD）
- [ ] 评分标准统一，每个文件都有"总分+具体问题+修改建议"
- [ ] 抽查1-2个低分文件，报告指出的问题确实存在
- [ ] 抽查1-2个高分文件，确实符合规范
- [ ] 审查报告已保存到 08-待审核区/审查报告-{日期}.md
- [ ] 工作日志已记录批次、数量、平均分和共性问题

## 边界（不做的事）
- 不大改已有内容——只指出问题
- 不因 `[待补充]` 多就打低分（那是正常未完成状态）
- 不直接修改被审查的文件

## 快速命令
```
用"质量审查 Agent"审查 {知识点文件路径}
```
