# 模板：网课上课思路提炼

> 📋 **使用场景**：当用户导入网课 PDF / Markdown / 课堂笔记 / 视频转写后，用本模板提取“老师怎么讲”，而不是机械摘抄知识点。  
> 🎯 **目标产物**：服务专题页、备课大纲、学生讲义、PPT、KP 教学视角的“上课思路数据库”。  
> ⚠️ **核心原则**：先保留原课讲授顺序，再判断哪些内容适合当前轮次，最后提出可回流到知识库的改造建议。
> 2026-06-18 起补充原则：网课提炼不仅记录“老师怎么讲”，还要回答“这些讲法以后如何被稳定抽进备课大纲、学生讲义、PPT 和教师口令”。

---

## 一、适用边界

本模板适合处理：

- 网课 Markdown 笔记
- 网课 PDF 转写材料
- 课堂实录整理
- 老师讲义中明显体现讲授顺序的内容
- 竞赛课程中的例题串、专题串讲、复习课思路

不适合直接处理：

- 只包含知识条目的教材正文：优先用 [[模板-资料提炼]]
- 已经整理好的正式备课大纲：优先用 [[模板-备课大纲]]
- 学生可直接阅读的课堂讲义：优先用 [[模板-新授课]]

## 八、本轮禁止提前展开内容

> ⚠️ 本节不是简单记录“原资料没讲的内容”。  
> 它记录的是：**对当前轮次而言不宜展开的内容**。来源可能有三类：
>
> 1. **原资料已经涉及，但当前轮次不宜展开**：例如原课为了体系完整提到高阶内容，但基础班只点到为止。
> 2. **原资料没有涉及，但相关 KP/专题中存在，且容易被 Agent 误塞进当前备课**：为了防止“把知识库已有高阶内容平铺进第一轮”。
> 3. **原资料作为悬念或后续承接出现**：课堂可用一句话回应，但不在本轮训练。

| 内容 | 来源类型 | 为什么暂不展开 | 放到哪个轮次/专题 | 若课堂被问到如何回应 |

## 十一、MD / PDF / minerU 识别边界

> 网课资料常见问题：Obsidian 中 MD 渲染不如 minerU，公式、图片、表格可能错位。因此提炼时必须区分“教学逻辑可直接识别”和“细节需复核”。

| 内容类型 | 是否可直接依赖 MD | 处理规则 |
|:---|:---:|:---|
| 讲课顺序、标题层级、概念出现先后 | 是 | 用于上课思路提炼主线 |
| 老师强调的易错点、例题出现位置 | 基本可以 | 可提取教学作用，但题面细节可后续复核 |
| 复杂公式、表格数据、长计算过程 | 不完全可以 | 标注“需公式/数据复核”，不要静默改写 |
| 图片、图像题、反应进程图、装置图 | 不完全可以 | 需要读取图片或回看 minerU/PDF 渲染版 |

推荐固定采用“两遍法”：

```text
第 1 遍：MD 结构提取
  抓讲课顺序、认知台阶、例题位置、轮次边界

第 2 遍：公式/图片/题面复核
  在正式备课大纲、学生讲义、PPT、KP 回流前，对照 minerU/PDF/图片
```

## 十二、Agent 提炼规则

### 11.1 必须做

- [ ] 先保留原课讲解顺序，再做知识库重组。
- [ ] 明确“原资料有无涉及”与“当前轮次是否适合展开”是两件事。
- [ ] 每个核心概念都要判断：第一轮必讲 / 点到为止 / 后续轮次。
- [ ] 例题不只摘题目，还要标注教学作用。
- [ ] 至少列出 3 个可回流目标：专题页 / KP / 备课大纲 / 学生讲义。
- [ ] 若正式回流到备课大纲，必须把本文件加入 `logic_sources`。

### 11.2 禁止做

- [ ] 不要把网课全文 OCR 机械塞进 vault。
- [ ] 不要把原课顺序直接等同于本知识库第一轮顺序。
- [ ] 不要因为原课讲了某内容，就默认基础班必须展开。
- [ ] 不要因为原课没讲某内容，就默认当前轮次不存在这个边界问题。
- [ ] 不要直接改正式 KP/专题页，除非用户确认“回流”。

## 十三、下一步建议

1. 若只是试点：先给用户审核，不直接回流正式笔记。
