# 模板-活跃任务卡

> **用途**：为 `00-首页/活跃任务/` 下的单个任务卡提供统一 frontmatter 与正文结构。
>
> **核心原则**：任务卡是 **Agent 的约束锚点**。创建任务卡 = 承诺”我要做这件事”。执行期间以任务卡 frontmatter 为导航，收工先更新任务卡，再由任务卡状态决定入口页同步需求。
>
> **约束机制**：
> - `status` 必须随时反映真实状态（active / blocked / completed），不允许”做完了但忘记改”
> - `task_type` 自动决定路由规则——Agent 不需要重读 SOP 来确认”做完要回写哪里”
> - `evidence` 必须在 Phase 4 填写，空 evidence = 任务未完成
> - [[活跃任务]] 聚合页每轮加载时会显示所有 active 卡，未更新的卡会被用户/下一位 Agent 发现

## Frontmatter 模板

```yaml
---
title: {任务标题}
type: 活跃任务卡
task_type: {任务类型}      # 路由分类：资料提炼 / 知识点填充 / 题库建设 / 内容优化 / 备课产出 / 答疑 / 系统维护 / 审计 / 其他
status: active              # active / blocked / completed / paused
priority: P1                # P1 / P2 / P3
area: {领域}                # 如：备课体系 / 知识点维护 / 脚本维护 / 首页系统
owner: Agent                # Agent / User / Agent+User
created: YYYY-MM-DD
updated: YYYY-MM-DD
completed:                  # 仅 completed 时填写
description: >              # 一句话描述本卡要完成什么（必填，用于快速路由判断）
  本卡要完成...
source_notes: "[[来源笔记]]"
related_notes:
  - "[[相关笔记1]]"
  - "[[相关笔记2]]"
evidence:
  - "[[工作日志/证据1]]"
  - "[[工作日志/证据2]]"
---
```

## 字段说明

| 字段 | 必填 | 含义 | 取值建议 |
|---|---|---|---|
| `title` | 是 | 任务名 | “动作 + 对象”命名 |
| `type` | 是 | 文件类型 | 固定 `活跃任务卡` |
| **`task_type`** | **是** | **任务类型** | **见下表——决定 Agent 路由规则** |
| `status` | 是 | 当前状态 | `active` / `blocked` / `completed` / `paused` |
| `priority` | 是 | 优先级 | `P1` / `P2` / `P3` |
| `area` | 是 | 所属领域 | 如：备课体系 / 知识点维护 / 脚本维护 / 首页系统 |
| `owner` | 是 | 责任方 | `Agent` / `User` / `Agent+User` |
| `created` | 是 | 建卡日期 | `YYYY-MM-DD` |
| `updated` | 是 | 最近更新 | 每次推进都更新 |
| `completed` | 否 | 完成日期 | 仅 `status: completed` 时填写 |
| **`description`** | **是** | **一句话描述** | **用于 Agent 快速判断任务范围+回写需求** |
| `source_notes` | 是 | 任务来源 | 触发此任务的入口文件 |
| `related_notes` | 是 | 相关资料 | 专题页 / KP / 模板 / 战略页等 |
| `evidence` | **必填(Phase 4)** | 证据链接 | 日志、产出文件、验证文件**——空 evidence = 任务未完成** |

### `task_type` 取值 → 路由规则

| task_type | Agent 读取顺序 | 默认提示词 | 收工必须同步 |
|:---|:---|:---|---:|
| `资料提炼` | 摘要+活跃→战略→模块关系清单 | [[Agent-1-资料提炼-轻量]] | 工作日志 + 活跃任务卡 |
| `知识点填充` | 摘要+活跃→战略 | [[Agent-2-知识点填充-轻量]] | 工作日志 + 活跃任务卡 |
| `题库建设` | 摘要+活跃→战略→模块关系清单 | [[Agent-3/4/5]] | 工作日志 + 活跃任务卡 |
| `内容优化` | 摘要+活跃→战略→模块关系清单 | [[Agent-7/8/9]] | 工作日志 + 活跃任务卡 |
| `备课产出` | 摘要+活跃→战略→模块关系清单→备课待办 | [[Agent-10-备课-轻量]] | 工作日志 + 活跃任务卡 + **备课待办** |
| `答疑` | 摘要（必要时） | [[Agent-11-答疑-轻量]] | 工作日志 |
| `系统维护` | 摘要+活跃→模块关系清单 | **升级完整 SOP** | 工作日志 + 活跃任务卡 + **状态摘要**（结论变化时）|
| `审计` | 摘要+活跃→战略 | [[Agent-9-健康检查-轻量]] → SOP | 工作日志 + 活跃任务卡 + **审计报告** |
| `其他` | 完整 Phase 0 | — | 询问用户 |

## 正文模板

```md
# {任务标题}

## 目标

一句话说明这张任务卡要完成什么。

## 完成标准

- [ ] 标准 1
- [ ] 标准 2

## 可关闭路径

- 路径 A：实际产出完成
- 路径 B：审计后接受“复用已有产物”

## 当前判断

- 为什么现在要做
- 当前卡点是什么

## 下一步

1. 下一步动作

## 关闭动作

- 完成后：`status: completed` + 填 `completed` + 填 `evidence`
- 被阻塞：`status: blocked` + 写明解除条件
- 暂停：`status: paused` + 写明恢复条件
```

## 更新规则

1. **开工前先建卡/认领卡**：Agent 开始任何任务前，必须确认有对应的任务卡，`status` 为 `active`，`task_type` 已正确填写
2. **推进中**：更新 `updated`，必要时补充”当前判断 / 下一步”
3. **已完成**：`status: completed` + 填 `completed` + **必须填 `evidence`**
4. **被阻塞**：`status: blocked` + 写清阻塞原因与解除条件
5. **暂停（非阻塞的主动暂停）**：`status: paused` + 写清恢复条件
6. **证据不可为空**：`evidence` 为空的 completed 卡视为”未完成”，下一位 Agent 应打回
7. 不删除旧任务卡：历史任务保留，用于回溯与审计
8. 若任务属于”资料导入 / 资料提炼 / 网课提炼”链条，完成标准中必须显式检查是否已补齐”备课抽料资产标记”
