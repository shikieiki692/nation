# 讲义重组标准流程

> **用途**：为"对已有讲义进行结构性变更"（插入新章节、合并外部内容、大段重编号等）提供标准化操作流程，避免遗漏和编号错乱。
> **适用场景**：
> - 在已有讲义中插入新的大节/小节
> - 将一份桥梁讲义/衔接讲义的内容合并入主线讲义
> - 对讲义进行大规模章节重编号
> - 合并多份讲义为一份
> **不适用**：纯内容增补（不改变章节结构）、格式清理（不涉及重编号）
> **提炼来源**：2026-06-30 分子结构基础-超级充实版 §一 化学键基础插入实战

---

## 一、四阶段流程总览

```
A. 内容审查 → B. 结构变更 → C. 编号同步 → D. 质量验收
     ↓              ↓              ↓              ↓
  缺口表确认    插入/合并/删除   全文件重编号    预检+交叉校验
```

每个阶段完成后必须通过对应的**检查点**才能进入下一阶段。

---

## 二、阶段 A：内容审查

> **目标**：明确"要变什么、为什么变、变了以后前后怎么接"。

### A1. 确认变更需求

| 问 | 答 |
|:---|:---|
| 这次重组的类型是什么？ | □ 插入新章节 □ 合并外部讲义 □ 大段重编号 □ 其他 |
| 变更的触发原因是什么？ | （内容缺口分析结果 / 用户要求 / 教学逻辑调整） |
| 参照了哪些教材/知识库资源？ | （列出 L1-L3 来源） |

### A2. 梳理内容缺口

1. 对比主框架教材（L1）和辅助教材（L2），列出缺失内容
2. 在知识库中搜索已有 KP、专题页、教学洞察，确认可复用资产
3. 输出**缺口表**：每条缺口写明"缺什么、从哪补、放在哪个位置"

### A3. 确认插入位置

- 明确新内容放在现有讲义的哪个章节之前/之后
- 确认新内容与前后章节的**衔接逻辑**（为什么要在这个位置）
- 如果涉及"吸收外部讲义"，确认被吸收讲义的哪些内容进入、哪些不进入

### ✅ 检查点 A

- [ ] 变更类型和原因已明确
- [ ] 缺口表已列出，来源可追溯
- [ ] 插入位置和前后衔接逻辑已确认
- [ ] 如涉及外部讲义吸收，已确认退役计划

---

## 三、阶段 B：结构变更

> **目标**：执行实际的内容插入、合并或删除操作。

### B1. 执行内容变更

- 按缺口表逐条执行插入/合并
- 每完成一个大节的插入，立即在该节内完成子节编号（如 `### 1.1` → `### 1.2`...）
- 如果合并了外部讲义的内容，在正文首次出现处添加来源注释：
  ```markdown
  > 以下内容整合自 [[过渡-化学键衔接]] §一~§三，已合并入本节。
  ```

### B2. 标记被吸收的外部讲义

如果本次重组将一份独立讲义的内容合并入主线讲义：

1. 在被吸收讲义的 frontmatter 中添加退役字段：
   ```yaml
   stage: deprecated
   deprecated_date: YYYY-MM-DD
   deprecated_reason: "核心内容已合并入 [[目标讲义路径]] §X.Y"
   superseded_by: "[[目标讲义路径]]"
   ```

2. 在被吸收讲义的标题下方添加醒目退役提示：
   ```markdown
   > [!warning] ⚠️ 已退役
   > 本讲义的**全部核心内容**已于 YYYY-MM-DD 合并入 [[目标讲义路径]] **§X.Y 标题**。
   > 后续使用请以目标讲义为准，本文件不再维护。
   ```

### B3. 中间检查：标题结构完整性

执行完所有内容变更后，立即检查标题层级是否完整：

```bash
# 列出所有标题行，检查层级连续性
grep -n "^## \|^### \|^#### " 讲义文件.md
```

检查项：
- `##` 章标题序号是否连续（一、二、三...）
- `###` 节标题序号是否连续（1.1, 1.2, 1.3...）
- 无跳号、无重复

### ✅ 检查点 B

- [ ] 所有内容变更已执行
- [ ] 被吸收讲义已标记退役（如适用）
- [ ] 标题层级结构完整、序号连续

---

## 四、阶段 C：编号同步

> **目标**：确保正文中的所有引用编号（图注、练习题、答案、内部交叉引用）与新结构一致。

### C1. 全文件图注编号校验

```bash
# 提取所有图注行
grep -n "^\*图 [0-9]" 讲义文件.md
```

检查项：
- 图注编号是否从 1 开始连续递增
- 无跳号、无重复、无中断
- 每张图注与上方的 `![[media/...]]` 一一对应

### C2. 练习题编号校验

```bash
# 提取所有练习题编号
grep -n "^\*\*[0-9]\+\." 讲义文件.md
```

检查项：
- 练习题编号是否从 1 开始连续递增
- 三个分区（🌱基础/🔥竞赛/🏆真题）各自编号连续
- 题目总数与分区标题中声称的数量一致

### C3. 答案编号校验

```bash
# 提取答案区编号
grep -n "答案" 讲义文件.md  # 先定位答案区
# 然后检查答案编号
```

检查项：
- 答案编号与练习题编号一一对应
- 如有编号映射变更（如原第 3-10 题→现第 5-12 题），在答案区头部写明映射注释

### C4. Frontmatter 同步

更新以下 frontmatter 字段：

| 字段 | 更新规则 |
|:---|:---|
| `image_count` | 与正文 `![[media/...]]` 总出现次数一致（注意：preflight 按出现次数统计，非唯一图片数） |
| `updated` | 更新为当天日期 |
| `last_audit` | 写明本次变更摘要 |
| `tags` | 如新增了知识主题，补充对应 tag |
| `serve_topics` | 如新增了服务主题，补充对应 topic |
| `related_notes` | 如新增了关联讲义/KP，补充对应 wikilink |

### C5. 内部交叉引用检查

```bash
# 搜索正文中引用章节号的地方
grep -n "§[一二三四五六七八九十]" 讲义文件.md
grep -n "§[0-9]" 讲义文件.md
```

检查项：
- 正文中提到"详见 §X.Y"的引用是否指向正确的新编号
- 学习目标中引用的章节是否与实际章节对应
- 延伸阅读/前置要求中的引用是否正确

### ✅ 检查点 C

- [ ] 图注编号 1-N 连续无中断
- [ ] 练习题编号连续、总数与声明一致
- [ ] 答案编号与练习题一一对应
- [ ] Frontmatter image_count/updated/last_audit 已更新
- [ ] 正文内的章节引用已全部校正

---

## 五、阶段 D：质量验收

> **目标**：通过工具校验和人工复核，确认重组后的讲义质量达标。

### D1. PDF 预检器

```bash
python "11-模板/scripts/pdf_preflight.py" "讲义文件路径.md"
```

必须通过的标准：
- **0 error**（WARNING 中的 LONG_CAPTION 如内容必要可接受）
- CAPTION_SEQUENCE 不报警（图注编号连续）
- IMAGE_COUNT_MISMATCH 不报警（image_count 与实际一致）
- STAR_MARKER 不报警（无星级残留）
- EMOJI_CALLOUT 不报警（无 emoji callout 残留）

### D2. 图片唯一性检查

```bash
# 提取所有图片文件名，检查是否有同图重复放置
grep -o "media/[^]|]*" 讲义文件.md | sort | uniq -d
```

如果输出非空，说明有重复图片。对每张重复图片判断：
- 如果是同一知识点不同位置需要→保留，两处图注应不同
- 如果是纯粹重复放置→删除重复的一处，替换为文字引用或交叉引用

### D3. 关联文件检查

| 场景 | 需要同步更新的文件 |
|:---|:---|
| 合并了外部讲义 | 被合并讲义需加退役标记（阶段 B2） |
| 新增了知识主题 | 对应专题页的 `related_notes` 是否需要反向添加 |
| 六大章节索引页 | `备课思路/新课大章节-*.md` 中的讲义映射表是否需要更新 |

### D4. 人工复核建议

- 快速浏览重组后的目录结构，确认"读起来逻辑通顺"
- 检查新插入内容前后的过渡段是否自然
- 确认深度边界声明是否仍然准确

### ✅ 检查点 D

- [ ] preflight 0 error
- [ ] 无同图重复放置
- [ ] 关联文件已同步更新
- [ ] 人工复核通过

## 六、快速参考卡

> 打印或收藏本节，作为每次重组时的 checklist。

### 重组前 5 分钟

1. 确认变更类型（插入/合并/重编号）
2. 列出缺口表 + 确认插入位置
3. 确认主框架教材和辅助教材来源

### 重组中

4. 逐节插入内容，每节完成后立即编子节号
5. 如吸收外部讲义→加退役标记
6. 标题结构完整性中途检查

### 重组后 10 分钟

7. 全文件图注编号校验（grep `^\*图`）
8. 练习题+答案编号校验
9. Frontmatter 同步（image_count/updated/last_audit/tags）
10. 正文内章节引用校正

### 验收

11. `pdf_preflight.py` 跑一遍（0 error）
12. 图片唯一性检查（grep + uniq -d）
13. 关联文件同步检查

---

## 七、典型踩坑与规避

| 踩坑 | 规避 |
|:---|:---|
| 答案编号与练习题编号不同步 | 阶段 C3 强制交叉核对 |
| 图注编号在插入点后重新从 1 开始 | 阶段 C1 用 grep 一次性检查全文件 |
| 被吸收讲义未标记退役，未来被误用 | 阶段 B2 强制加退役标记 |
| image_count 与实际不一致 | 阶段 C4 用 preflight 自动校验 |
| 正文中"详见 §X"引用指向旧编号 | 阶段 C5 用 grep 搜索所有 § 引用 |
| 同一图片在不同位置重复放置 | 阶段 D2 用 uniq -d 检查 |
