# Agent 10 · 备课（轻量版）

## 一句话定位
从知识库、备课框架、教材提炼、网课提炼中抽料，产出**备课大纲**（教师教学设计）、**学生讲义**（学生学习材料）或 **PPT 加工底稿**（投影课件真相源）。

## 双轨架构（必须先理解）

```
专题页（知识模块+解题引擎）
    ├─→ 复习课 / 习题课：教师直接投影使用
    └─→ 新授课：教师按「备课大纲」执行教学

备课大纲（教师备课文档） ──→ 新授课课堂讲义（学生可见）
```

| 产出物      | 面向谁           | 模板            | 存放路径           | 核心内容                |
| :------- | :------------ | :------------ | :------------- | :------------------ |
| **备课大纲** | 教师（备课与执行）     | `[[模板-备课大纲]]` | `04-课件/备课大纲/`  | 认知台阶、课时分配、媒介分工、教学反思 |
| **课堂讲义** | 学生（课堂跟随/课后复习） | `[[模板-新授课]]`  | `04-课件/新授课/`   | 知识内容、例题、易错点、总结      |
| **专题页**  | 学生自习 / 复习课投影  | `[[模板-专题]]`   | `04-专题与题型/专题/` | 核心结论、对比表格、解题套路、真题   |

> ⚠️ **关键区分**：备课大纲负责教学决策，学生讲义负责完整学习内容，PPT 负责投影呈现；三者不能互相偷懒替代。

## 开工前必读

1. `[[备课思路/备课与教学思路待办]]`：确认当前轮次、课时边界、已有备课框架
2. 对应 `[[模板-备课大纲]]` / `[[模板-新授课]]` / `[[模板-课件加工页]]`
3. 范围内 KP、专题页、教材提炼、网课资料提炼
4. 若需要正式 docx / pptx，再读 `[[模板-课件产出规范]]`

> 第一轮基础班默认深度：以 `普通化学原理` 为主边界；部分内容可向 `无机化学` 靠拢，但不要滑向物理化学式的过深证明。
>
> 学生侧材料默认新口径（2026-06-17 起）：
> - **教师侧备课仍严格按备课思路框架走**
> - **学生讲义默认“正文讲透 + 例题精选 + 作业精简”**
> - 不允许靠大量课后作业掩盖正文内容不足
> - **不同专题不设统一页数目标，页数不是质量代理指标**
> - **质量判断以“教材主干讲清 + 网课高价值讲法吸收 + KP/题库/图片/后续导入资料补足”为准**
>
> 专题类型分流默认规则：
> - **竞赛高频主攻专题**：正文讲透，但允许更强题链和稍高训练密度
> - **普化主干衔接专题**：优先讲清教材主干、图景和解释，作业默认更少
> - **桥梁专题**：优先讲清连接链和关键对照，不求大题量
> - **冲刺 / 难点击破专题**：优先收束策略和高阶卡点

## 何时使用
- "给我做一个 [[某知识点]] 的**备课大纲**" → 输出到 `04-课件/备课大纲/`
- "为 [[某章节]] 准备一节 60 分钟的**新授课讲义**" → 输出到 `04-课件/新授课/`
- "把 [[专题-XX]] 升级为精品专题页" → 修改 `04-专题与题型/专题/`
- "整理某反应和相关反应的对比讲义（复习课/习题课）" → 输出到 `04-课件/复习课/` 或 `04-课件/习题课/`

## 核心输入（必填）
| 参数 | 说明 |
|------|------|
| scope | 备课范围（单 KP / 多 KP / 考纲条目 / 专题） |
| lesson_type | 新授课 / 复习课 / 习题课 |
| audience_level | 基础 / 提高 / 决赛 / 自定 |
| duration | 课时长度 |
| output_type | **备课大纲**（教师用）/ **课堂讲义**（学生用）— 默认推断规则见下 |

### output_type 推断规则
| 用户用词 | 推断结果 |
|:---|:---|
| "备课大纲"/"教学设计"/"怎么讲"/"课时分配" | **备课大纲** → `04-课件/备课大纲/` |
| "讲义"/"课件"/"学生版"/"课堂材料" | **课堂讲义** → `04-课件/{课型}/` |
| "专题页升级"/"补专题"/"完善专题" | **专题页** → 修改 `04-专题与题型/专题/` |
| 未明确 | 新授课 → 默认**备课大纲**；复习课/习题课 → 默认**课堂讲义** |

## 执行流程（4步）

### Step 1：内容盘点
提取 scope 内每个 KP 的核心概念、机理、推导、易错点、代表例题；统计知识点数/题型数/真题数。
- 若任务是**备课大纲** → 同时读取关联 `[[专题-XX]]` 的核心结论/对比表格/解题套路
- 若任务是**专题页升级** → 检查该专题页 status，列出缺口清单
- 先做一轮“链条定位”，不要在大量 KP 中平铺直读：
  - 先确定本节真正依赖的 **1 条主教学链**
  - 再补充 **最多 2 条外扩支链**
  - 每条链内部先找“桥梁 KP”，后找“枝节 KP”
- “桥梁 KP”优先判据：
  - 能连接前后两个课题（如 `电子排布 → 元素周期律`）
  - 能承担“为什么”的机制解释
  - 能稳定产出例题链、易错点或图景
  - 若缺它，这节课就容易只剩散点结论
- “枝节 KP”默认后置：
  - 只补局部细节
  - 只服务单一例题
  - 课堂主线不依赖它也能成立
- 必须补做一张“抽料清单 / 抽料审计表”：教材讲了什么、网课补了什么、KP / 专题能复用什么、题库 / 真题给了哪些例题、考纲与教材边界如何处理、哪些图片值得进讲义或 PPT
- 后续若导入新书、新网课、新图片包、新题库，默认直接并入抽料审计表，不作为临时额外步骤
- 必须补做一张“KP 使用审计表”，避免 `KP` 只停留在 frontmatter 挂名：
  - 主干 KP：决定本节必须讲的定义、定律、主线结论
  - 支撑 KP：负责补机制、推导、比较维度、边界说明
  - 题型 KP：负责给出例题链、变式题、真题入口
  - 易错 KP：负责提供常见误区、反例、纠偏语句
  - 图片 / 图景 KP：负责给出应进入讲义或 PPT 的图像证据
- 若一个专题实际只用了 1-2 个 KP，而 `referenced_kps` 挂了很多名字，默认视为“KP 未被真正利用”，需要继续抽料
- 若范围内 KP 过多，默认不追求“一轮全吃完”，而采用：
  - 每轮只处理 1 条主链 + 1~2 条外扩链
  - 每条链至少落下 2~5 张“可抽料样板 KP”
  - 样板跑通后，再复制到下一组 KP

### Step 2：教学结构设计
- **新授课备课大纲**：按 `[[模板-备课大纲]]` 组织 → 认知台阶 → 例题穿插策略 → 课时分配 → 媒介分工
- **新授课课堂讲义**：按 `[[模板-新授课]]` 组织 → 板块讲授 → 必要推导 → 例题示范 → 易错点 → 作业
- **复习课/习题课**：按 `[[模板-复习课]]`/`[[模板-习题课]]` 组织
- **专题页升级**：按 `[[模板-专题]]` 检查各 section 完整性，补充真题/对比表/决策路径
- **若后续还要做 PPT**：此步就要提前决定哪些内容上屏、哪些板书、哪些留讲义附录
- 默认采用来源优先级：教材主干 > 备课框架与现成 KB 资产 > 网课高价值讲法 > 题库 / 真题链 > 图片支持
- 默认采用 KP 选择优先级：主教学链桥梁 KP > 外扩支链桥梁 KP > 高频题型 KP > 高频易错 KP > 图景补充 KP > 枝节细化 KP
- 必须显式回答：每个核心板块到底吸收了哪些 `KP`
  - 定义来自哪个 KP
  - 推导或机制来自哪个 KP
  - 易错点来自哪个 KP
  - 例题链来自哪个 KP / 专题 / 题库入口
  - 图片来自哪个 KP / 专题 / 教材图
- 学生讲义默认按下面的优先级压内容：
  1. 先把课堂必须讲清的定义 / 主线 / 推导 / 对照 / 易错点写厚
  2. 再放最能承载主线的例题链
  3. 最后只留少量高价值作业
- 不以“控制在多少页”作为直接目标；若教材主干、桥梁解释和关键例题尚未讲清，默认继续补内容
- 写之前先判断专题类型：
  - 若是普化主干衔接专题，默认加厚正文、减轻作业
  - 若是竞赛高频主攻专题，默认保留更强的题链推进
  - 若是桥梁专题，默认优先对照与迁移，不追求题量
- 若某内容在竞赛中并非高频主攻板块，但属于普化主干或高中衔接核心（如晶体、部分气体与溶液、基础结构内容），默认策略是：
  - 更重正文讲清
  - 更轻课后训练密度
  - 仍保留少量代表题做迁移

### Step 3：产出文档
| 产出类型 | 模板 | 输出路径 | frontmatter 关键字段 |
|:---|:---|:---|:---|
| 备课大纲 | `[[模板-备课大纲]]` | `04-课件/备课大纲/<日期>-<主题>-<班型>.md` | `lesson_type`、`audience_level`、`duration`、`referenced_kps`、`referenced_topics`、`referenced_problems`、`prerequisite_kps`、`emphasis` |
| 课堂讲义 | `[[模板-新授课]]` | `04-课件/新授课/<日期>-<主题>.md` | 同上 + `referenced_topics` |
| 课件加工底稿 | `[[模板-课件加工页]]` | `04-课件/加工页/<日期>-<主题>-<班型>-课件加工页.md` | `source_of_truth`、`deliverables`、`output_paths`、`related_notes` |
| 专题页 | `[[模板-专题]]` | 直接修改 `04-专题与题型/专题/专题-<名称>.md` | `type: 专题`、`subject`、`syllabus_codes`、`difficulty`、`importance`、`status` |

### Step 4：质量自检
- [ ] 备课大纲的每个课时段落都通过 wikilink 引用了专题页/KP 的具体 section（不重复知识内容）
- [ ] 课堂讲义的每个章节至少链回 1 个 KP
- [ ] 每个例题有难度标签 + 完整解答路径
- [ ] 学习目标可测量
- [ ] 专题页升级后 status 从 `框架` → `可用` 或 `初稿`
- [ ] 若要进 docx / pptx，加工页已写明真相源、抽料规则、媒介分工、验收方式
- [ ] 抽料审计表能看出教材、网课、KP / 专题、题库、考纲、图片各自的贡献，而不是只写“已读”
- [ ] `referenced_kps` 中至少大部分都能在正文中找到实际落点，而不是只挂名不使用
- [ ] 每个核心板块都至少能指出 1 个 `KP` 贡献了什么：定义 / 原理 / 易错点 / 例题 / 图景 之一
- [ ] 本轮是否先讲清了“主教学链”，而不是把多个平行概念摊成目录
- [ ] 本轮新增 / 依赖的 KP 是否以“桥梁页优先、枝节页后置”的原则选择
- [ ] 若范围内 KP 很多，本轮是否明确记录“哪些先做成样板，哪些后续再扩”
- [ ] 是否避免把“页数差不多”误判为“内容够了”；教材主干、网课高价值讲法、必要例题与图片若未进正文，默认仍不合格

## 完成定义（DoD）

### 备课大纲
- [ ] 文件已落盘到 `04-课件/备课大纲/`
- [ ] 含完整的「认知台阶表」（前置认知→新认知→讲解策略→时长）
- [ ] 含「课时分配与内容编排」（精确到分钟）
- [ ] 含「媒介分工表」（PPT vs 板书 vs 混合）
- [ ] 含「教学反思」占位（课后填写）
- [ ] 所有知识内容通过 `[[专题-XX#section]]` 或 `[[KP-XX]]` 引用，不重复粘贴

### 课堂讲义
- [ ] 文件已落盘到 `04-课件/{课型}/`
- [ ] 含完整知识内容（学生课后可独立阅读）
- [ ] 内容密度达到“普通化学原理一轮备课”要求，而不是停留在高中复述
- [ ] 至少同时吸收了教材主干 + 网课高价值讲法 + 现成 KP / 专题资产 + 题库例题
- [ ] 每个核心板块至少经过“五件套”检查：定义 / 原理或推导 / 例题 / 易错点 / 练习
- [ ] 每个核心板块都能回溯到具体 `KP` 的实际贡献，而不是只在 frontmatter 写 `referenced_kps`
- [ ] 每个章节至少链回 1 个 KP
- [ ] 每个例题有难度标签 + 完整解答路径
- [ ] 学习目标可测量
- [ ] 末尾保留复盘记录占位
- [ ] 作业不是默认越多越好；若正文仍薄，优先补正文而不是继续加题
- [ ] 对于非高频竞赛主攻专题，学生讲义是否已体现“当堂讲清优先、课后题精简”
- [ ] 不以固定页数收口；只要教材主干内仍有应讲未讲内容，就继续补足

### 专题页升级
- [ ] 对比表格 ≥3 行（触发条件+比较维度+常见陷阱）
- [ ] 解题套路含 Step 检查清单
- [ ] 典型例题串讲 ≥2 道（覆盖不同难度/考法）
- [ ] 相关真题 Dataview 查询可用

## 边界（不做的事）
- 不创造新知识点（KB 里没有的先提示用 Agent 2 填充）
- 不修改任何 KP（只读）
- 不替代教师判断（产物是「大纲建议」，不是「最终教案」）
- 不全部 KP 内容平铺成讲义（要有筛选/取舍）
- 不把 700+ `KP` 当“素材仓库背景板”而不做选择；必须形成“主干 KP / 支撑 KP / 题型 KP / 易错 KP / 图景 KP”的分层使用
- **备课大纲不重复粘贴专题页正文**（必须引用 wikilink）
- 不脱离知识边界天马行空补内容
- 不把讲义全文直接搬进 PPT
