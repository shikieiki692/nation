# Agent 提示词：PPT 课件生成（v1.0）

> **用途**：Agent 将备课内容（大纲 + 讲义 + 专题页）转换为结构化 PPT 课件时的内容层规范。
> **配套技术工具**：`[[11-模板/pptx-toolkit/README]]`
> **前置要求**：必须先读取备课大纲和新授课讲义，禁止凭空构造内容。

---

## 输入清单（Phase 0 必读）

生成 PPT 前，必须按优先级读取以下文件：

| 优先级 | 文件路径模板 | 提取内容 |
|:---|:---|:---|
| P0 | `04-课件/备课大纲/YYYY-MM-DD-主题-班型.md` | 认知台阶、课时分配、**媒介分工表** |
| P1 | `04-课件/新授课/YYYY-MM-DD-主题-班型.md` | 核心结论、知识内容（含展开）、例题、易错点 |
| P2 | `03-专题/专题-{主题}.md` | 对比表格、解题套路、典型例题串讲、关联 KP |
| P3 | `11-模板/pptx-toolkit/模板-PPT生成.js` | 技术模板（复制后填写内容） |

> ⚠️ **红线**：未读备课大纲就生成 PPT → 必定出现"逻辑跳跃"和"内容过浅"。

## 幻灯片类型规范（六类模板）

### 类型 1：过渡页（Transition）

**触发条件**：当 PPT 从一个理论/方法切换到下一个时，两者存在"进化关系"或"适用范围差异"。

**内容要求**：
- 大标题：`"为什么 XX 不够用了？"` 或 `"从 XX 到 YY"`
- 中间：旧理论的 2 个典型场景（✅ 能解释）+ 1 个反例（❌ 不能解释）
- 底部：进化动机 → `"需要更广义的定义"`

**最小内容量**：1 个旧理论应用 + 1 个反例 + 1 个进化动机。

**为什么重要**：没有过渡页，学生不知道"为什么要学新理论"，知识变成孤立碎片。

---

### 类型 2：核心结论页（Core Conclusion）

**触发条件**：每个理论/方法的第一页；每 20min 课时的知识锚点。

**内容要求**：
- 上方：**1 句话本质定义**（直觉级，不是公式）
- 中间：核心公式（LaTeX，走 `formulas` 注册表）
- 下方：**3 个决策检查点**（"什么时候用这个理论？"）

**最小内容量**：1 个本质定义 + 1-2 个公式 + 3 个检查点。

**示例**（Brønsted 理论）：
```
本质定义：酸和碱的本质是质子（H⁺）的转移，不是物质的固定属性。
决策检查点：
  1. 反应中是否有质子（H⁺）的给出或接受？
  2. 溶剂是否是质子性溶剂（水、醇、液氨）？
  3. 需要判断共轭酸碱对的相对强弱时？
```

---

### 类型 3：展开页（Elaboration）

**触发条件**：核心结论后，需要详细展开边界条件、适用范围、对比、机理。

**内容要求**：
- **禁止大段文字罗列**。必须用结构化形式：
  - 对比表（≥3 行，必须含"触发条件"或"适用范围"列）
  - 决策流程图（Mermaid 或文字箭头）
  - 边界条件清单（✓ 适用 / ✗ 不适用 / ⚠️ 注意）
- 每个要点配 **1 个最小示例**（具体数字或化合物）

**最小内容量**：≥2 个结构化区块 + ≥1 个示例。

**反例**（❌ 不要这样做）：
```
❌ "Brønsted 理论的优点是适用范围广，缺点是定义依赖质子，
     Lewis 理论的优点是不依赖质子，缺点是定义过宽..."
     （文字堆砌，学生看完记不住）
```

**正例**（✅ 应该这样做）：
```
✅ 表格：
| 维度 | Brønsted | Lewis |
|:---|:---|:---|
| 酸的定义 | 质子给体 | 电子对受体 |
| 能解释 NH₃ | ✓（接受质子） | ✓（提供孤对电子） |
| 能解释 BF₃ | ✗（无质子转移） | ✓（接受孤对电子） |
| 适用场景 | 质子溶剂中的反应 | 所有配位/氧化还原反应 |
```

---

### 类型 4：例题页（Example）

**触发条件**：讲义中的每道例题。

**强制拆分**：每道例题必须拆成 **2-3 页**，禁止一页直接给答案：

**页 A：审题与标注**
- 题干（精简版，删除无关背景描述）
- 关键数据用高亮背景框（`background: #FFF3CD` 类黄色）
- 页脚标注：对应 KP `[[...]]`、真题 `[[...]]`（文本形式，不要 wikilink）

**页 B：思路路径**
- 决策树或分步箭头（"看到 ___ → 想到 ___ → 使用 ___"）
- ⚠️ 常见错误路径用虚线/灰色标注（"如果这里用了 ___, 会错在 ___"）

**页 C：计算与反思**
- 分步计算，每步带单位
- 最终答案用高亮框
- 底部 3 行：
  - `🔑 本题关键：{1 句话}`
  - `⚠️ 易错点：{1 句话}`
  - `🔗 类似题型：{1-2 个关联方向}`

**最小内容量**：题干 + ≥3 步思路 + 分步计算 + ≥2 行反思。

---

### 类型 5：陷阱识别页（Pitfall）

**触发条件**：讲义 §六 易错点，或例题中发现的典型错误。

**内容要求**：
- 左侧：❌ 错误做法（红色背景或删除线样式）
- 右侧：✅ 正确做法（绿色背景或高亮样式）
- 中间：为什么错？（1 句话解释**根本原因**，不是"你记错了"）

**最小内容量**：1 个错误示范 + 1 个正确示范 + 1 个根本原因。

---

### 类型 6：关联页（Connection）

**触发条件**：每节课结束前，或一个完整理论模块结束后。

**内容要求**：
- 知识网络位置图：前置依赖 → **本节** → 后续延伸
- 关联 KP 列表（3-5 个，文本形式）
- 高频真题编号（2-3 道）
- 专题页名称

**最小内容量**：≥2 个前置 + ≥2 个后续 + ≥3 个 KP + ≥2 道真题。

**为什么重要**：学生课后复习时，知道"这个知识点在知识库的哪里"，知道"该去哪里找更多题"。

## 认知台阶检查表（生成后自检）

`slides` 数组填写完毕后，逐条检查：

- [ ] **第 1 页**是否回答了"这节课要解决什么问题？"
- [ ] **理论之间**是否有过渡页（或过渡段落）解释"为什么需要新理论"？
- [ ] **每个公式**出现时，是否先给了"什么时候用这个公式"的检查点？
- [ ] **每道例题**是否拆成了"审题 → 思路 → 计算"三页？
- [ ] **是否有至少 1 页**专门讲"常见错误及为什么错"？
- [ ] **最后 1 页**是否给出了知识网络链接（KP、真题、专题页）？
- [ ] **媒介分工**是否正确：PPT 只放结构化内容，长推导留给讲义/板书？

## 媒介分工提醒（PPT vs 板书 vs 讲义）

| 内容类型 | 放哪里 | 原因 |
|:---|:---|:---|
| 结构化对比表 | **PPT** | 板书抄太慢，PPT 视觉化记忆 |
| 复杂公式（LaTeX） | **PPT** | 渲染比手写清晰，格式统一 |
| 决策流程图 | **PPT** | 逻辑关系一目了然 |
| 例题题干 | **PPT** | 节省抄题时间 |
| 长段推导过程（>5 步） | **讲义/板书** | PPT 翻页快，学生来不及抄 |
| 需要实时修改的草图 | **板书** | 灵活性强 |
| 纯文字定义复述 | **讲义** | PPT 应结构化，不应堆砌文字 |
| 例题的详细计算草稿 | **讲义** | 学生课后对照，课上跟思路即可 |

> **原则**：PPT 是"视觉锚点"，不是"讲义的投影"。如果一页 PPT 上的文字超过 80 字，考虑拆页或移到讲义。

## 内容深度标准

### 最小深度（从"骨架"到"可用"的门槛）

| 幻灯片类型 | 浅（不合格） | 达标（最小可用） | 深（优秀） |
|:---|:---|:---|:---|
| 过渡页 | 直接列两个理论名字 | 旧理论 1 个应用 + 1 个反例 + 进化动机 | 配具体化合物/反应方程式说明反例 |
| 核心结论 | 只给公式，没有文字解释 | 本质定义 + 公式 + 3 个检查点 | 每个检查点配 1 个最小示例 |
| 展开页 | 大段文字描述优缺点 | ≥2 个结构化区块 + ≥1 个示例 | 对比表 + 决策图 + 边界条件 + 例题验证 |
| 例题页 | 直接给答案 | 审题 + 3 步思路 + 计算 + 反思 | 思路页含错误路径标注，反思含变式延伸 |
| 陷阱页 | "注意不要记错" | 错误示范 + 正确示范 + 根本原因 | 配真题中该错误的失分统计或典型场景 |
| 关联页 | "本节内容与前面有关" | 2+ 前置 + 2+ 后续 + 3+ KP + 2+ 真题 | 知识网络图（Mermaid 或文字层级） |

## 技术执行流程

```
1. 读取 P0/P1/P2 源文件，提取素材
2. 按六类幻灯片规划 slides 数组结构
3. 复制模板：cp "11-模板/pptx-toolkit/模板-PPT生成.js" "04-课件/PPT/主题-班型.js"
4. 填写内容：OUTPUT_PATH、formulas（LaTeX 注册表）、slides（六类幻灯片）
5. 运行生成：node "04-课件/PPT/主题-班型.js"
6. 自检：对照"认知台阶检查表"逐项确认
7. 输出验证：检查 formula-cache/、PPT 文件大小、缩略图视觉检查
```

## 常见问题（FAQ）

**Q：备课大纲里没写认知台阶，怎么办？**
A：退回到 Phase 0，要求用户补充大纲，或 Agent 自行从讲义中提取逻辑线并标注为"⚠️  inferred"。

**Q：讲义内容太多，PPT 放不完？**
A：按媒介分工表筛选——结构化内容进 PPT，长推导进讲义，例题只放题干+关键思路。

**Q：专题页是空的（status: 框架），还能关联吗？**
A：关联页中标注为"待填充"即可，同时记录到工作日志 follow-up，提醒后续填充专题页。

**Q：没有真题可关联？**
A：关联页可以只写 KP 和前置/后续，真题部分标注"待补充"。但最小可用标准是至少 2 道真题，如果没有，需在 Phase 4 工作日志中记录为缺口。
