# Agent 1 · 资料提炼（轻量版）

## 一句话定位
将教材/书籍/讲义类资料提取为结构化提炼笔记，并为后续备课抽料提供可直接复用的内容资产入口。

## 何时使用
- 新导入一本教材或参考书，需要提炼为笔记
- 已有提炼笔记需要补充或更新

## 核心输入（必填）
| 参数 | 说明 |
|------|------|
| 源文件路径 | 06-外部资料导入/ 下的 Markdown 文件 |
| 模式 | 每章模式（默认）或 总索引模式 |
| 章节范围 | 要提炼的章节或全书范围 |

## 执行流程（4步）
1. **选模式**：判断用「每章模式」（常规教科书）还是「总索引模式」（跨章主题书/论文集）
2. **对照边界**：先看教材主干，再对照考纲；考纲用于校边界，不机械删去所有考纲外但教学主干中稳定存在的内容
3. **结构化提炼**：按模板提取概念、例题、讲解脉络、踩坑回流，并标明哪些内容将来可用于备课抽料
4. **输出文件**：按模式存入对应目录，同时保证后续备课 Agent 能从中读出教材主干、例题、图片、误区、教学顺序这些资产

## 完成定义（DoD）
- [ ] frontmatter 含 `version`；若模板仍保留兼容字段，则 `template_version` 与 `version` 一致；同时包含 `source_type`、`subject`、`syllabus_codes`
- [ ] 「讲解脉络」段非空（至少3个子项）
- [ ] 例题表含「考点抽象」列且每行非空
- [ ] 「踩坑回流」4个问题都有答案（"无"也算）
- [ ] 图片已改为 wiki-link 格式
- [ ] 已明确本资料可为后续备课提供哪些资产：教材主干、桥梁解释、例题链、易错点、图片/图景、课堂讲法
- [ ] 未把“篇幅不长”误判成“价值不高”；真正重要的是可否补强后续备课内容厚度

## 边界（不做的事）
- 不改动原书公式和化学方程式
- 不自行添加原书没有的内容
- 不混用模式（一本书定了模式就不要中途切换）
- 不把"考点抽象"填成题面复述
- 不把资料提炼做成“只存档、不服务备课”的静态摘要
- 不把考纲当成唯一删减依据；教材主干里对课堂理解有价值的内容应保留并标注边界

## 快速命令
```
用"资料提炼 Agent"处理 {源文件} 的 {章节范围}
```
