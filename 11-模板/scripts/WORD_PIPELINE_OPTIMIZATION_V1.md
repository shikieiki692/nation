# Word 讲义管线优化清单 v1

## 一句话目标

把当前 `Markdown → Pandoc → docx → 后处理` 的讲义导出链路，收成一条**稳定、可维护、可预检、公式可编辑、排版接近教材成品**的 Word 管线。

当前主管线入口：

- `11-模板/scripts/build-all-handout-docx.py`
- `11-模板/scripts/docx_utils.py`
- `11-模板/scripts/templates/custom-reference.docx`
- `11-模板/scripts/render-docx.ps1`

统一输出目录：

- `00-首页/学生讲义Word/`

## 当前架构

```text
Markdown 源讲义
  → 预处理（frontmatter / wikilink / 图片引用 / callout / 公式兼容）
  → Pandoc 转 docx（LaTeX 数学模式 → OMML）
  → python-docx 后处理（字体 / 页边距 / 图注 / 页码 / 图片缩放）
  → 可选渲染预览（render preview）
```

这条路线本身是对的，后续重点不是“换路线”，而是把规则再收紧。

## 总体判断

当前 Word 管线已经能稳定出成品，但还存在 4 个结构性问题：

1. 样式还不够“样式驱动”，后处理里仍有较多 run 级补救。
2. 公式虽然能转成 OMML，但缺一层严格的源 Markdown 预检。
3. 图表排版已经有基础，但离“教材感”还差一层布局规则。
4. 字体口径需要彻底统一，避免不同 agent、不同讲义写出不同规范。

## 字体总原则

这是后续 Word 管线的硬规则：

1. 中文正文：`SimSun`
2. 中文标题/表头：`SimHei`
3. 图注：`FangSong`
4. 英文、数字、变量、单位、化学式中的西文部分、一般数学符号、一般标点符号：`Times New Roman`
5. 只有“非常特殊、必须保留中文风格”的中文符号，才允许跟随中文字体

落地解释：

- 不要再出现“正文是宋体，但括号、等号、英文单位、箭头、数字跟着宋体跑”的情况。
- 非中文主体内容，默认都应优先落到 `Times New Roman`。
- 如果一个 run 同时含中文和西文，目标效果仍然应是“中文走中文字体，西文/数字/一般符号走 TNR”。

## 排版优化方向

### 1. 样式驱动优先

下一版不要继续主要依赖“遍历所有 run 再补字体”，而要把以下样式先正式做进 `custom-reference.docx`：

- `Handout Title`
- `Handout Subtitle`
- `Heading 1`
- `Heading 2`
- `Body Text`
- `Methodology Block`
- `Mistake Block`
- `Figure Caption`
- `Table Caption`
- `Answer Block`
- `Table Body`

目标是：

- Pandoc 先把结构转出来
- 后处理主要负责“套样式”和“少量修复”
- 不再依赖大面积硬改局部字体

### 2. 标题与分页控制

建议加上这些规则：

- `Heading 1` 前可选分页
- 标题段落 `keep with next`
- 图注与图片绑定，避免一页底部只剩图注或只剩图片
- 例题标题与其首段/表格/图片尽量不拆页

### 3. 表格排版

当前“整表全居中”不够像教材，下一版建议：

- 数值列居中
- 说明列左对齐
- 表头黑体加粗
- 表格正文宋体
- 西文/数字/符号仍强制 `Times New Roman`
- 长表允许减小字号，但不要低于可读阈值

### 4. 图片布局

当前已有图片缩放与居中规则，但下一版建议补上：

- 支持两张小图并列
- 长图自动限制高度，避免一图占整页
- 图注统一单独样式，不再只靠“文本以图/表开头”识别
- 图注过长时自动告警

## 公式渲染优化方向

## 核心原则

Word 成品中的公式应尽量保持：

1. 可编辑
2. 结构正确
3. 上下标统一
4. 标准态、反应下标、平衡常数下标等写法统一

当前路线是 Pandoc 把 LaTeX 数学模式转换成 OMML，这条路线继续保留，不改成“公式图片化”。

## 源 Markdown 公式规范

后续讲义统一遵守下面这套规则：

1. 简单数字下标可以用 Unicode
2. 只要涉及字母下标、反应下标、标准态上标、复杂分式、根式、矩阵、求和、积分，统一进入数学模式
3. 永远不要出现裸写的 `K_p`、`K_c` 这类下划线格式
4. `ΔGθ`、`ΔrHθ` 这类标准态符号统一为 `\theta` 或 `θ` 体系，不再使用 `\circ`
5. 化学式如果进入数学模式，必须采用当前管线可兼容的写法，避免残留未处理的 LaTeX 宏

建议直接固化几条高频规范：

- `$K_{\mathrm{p}}$`
- `$K_{\mathrm{c}}$`
- `$\Delta G^\theta$`
- `$\Delta_{\mathrm{r}}H^\theta$`
- `$\sqrt{\cdots}$`
- `$\frac{\cdots}{\cdots}$`

## Word 公式显示规则

建议继续保留并强化：

- 行内短公式：与正文协调
- 显示公式：统一比正文略大
- 长行内公式必要时上调字号
- 同一讲义内部不要混用多套上标/下标风格

## 预检器 v1 范围

这是最值得优先落地的一步。

当前实现落点：

- 主脚本：`11-模板/scripts/build-all-handout-docx.py`
- 运行方式：`python build-all-handout-docx.py --precheck-only`
- 默认行为：正常导出 docx 前自动执行；若存在 `ERROR` 级问题则直接阻断转换

预检器应在 docx 生成前检查这些问题：

1. 残留裸下划线公式，如 `K_p`、`v_max`
2. `\circ` 被误用在标准态符号位置
3. 残留 `\ce{}`、`\overset{}`、`\underset{}`、`\displaylines{}` 等需预处理宏
4. 数学模式外出现高风险公式片段
5. 图注中残留复杂公式
6. 生成后 docx 内 `oMath`/`oMathPara` 数量异常少
7. Markdown 中公式块很多，但 Word 中几乎没有公式对象

预检输出应按严重级别分 3 层：

- `ERROR`：高概率导致格式错误，阻止生成
- `WARN`：可以生成，但需要人工复核
- `INFO`：风格提醒，不阻断

## 建议新增的 Word 专项审计项

除现有验证外，建议补这些检查：

1. 统计图片数、图注数，检查是否明显不匹配
2. 检查是否有“图注很长但图片很小”的失衡情况
3. 检查表格是否出现极长单元格未换行
4. 检查标题层级是否超过 Word 成品可接受深度
5. 检查是否有过深列表嵌套
6. 检查是否有疑似教师口吻残留

## 参考模板优化项

`custom-reference.docx` 下一轮建议补齐这些内容：

1. 标题样式矩阵
2. 图注/表注专用样式
3. 例题/答案区专用样式
4. 表格默认段前段后距
5. 目录样式
6. 页眉页脚基础样式

如果这一步做好，`docx_utils.py` 的后处理复杂度可以明显下降。

## 实施顺序

建议按下面顺序推进，不要一口气全改：

1. 先做 `Word 公式预检器 v1`
2. 再做 `reference docx` 样式矩阵
3. 再做图表布局增强
4. 最后收缩 `docx_utils.py` 的 run 级补丁逻辑

这样做的好处是：

- 先把最容易返工的公式问题挡在前面
- 再逐步把格式控制权从“脚本硬修”迁到“模板样式”

## 本轮明确约束

从这版方案开始，Word 管线默认遵守以下约束：

1. 不为了“看起来像公式”而把公式转成图片
2. 不允许 `K_p`、`K_c` 这类裸下划线写法继续扩散
3. 除非常特殊的中文符号外，其余符号统一优先 `Times New Roman`
4. 图注中避免复杂数学表达式
5. 样式问题优先在模板层解决，脚本层做兜底

## 结论

下一阶段 Word 管线不需要推倒重来，重点是“收规则、立预检、强样式、稳公式”。  
只要把这 4 件事做好，后续其他讲义扩写到“超级充实版”时，Word 成品的一致性会明显提升。
