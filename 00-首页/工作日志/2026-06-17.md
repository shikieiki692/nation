# 2026-06-17 工作日志

## 📅 2026-06-17 备课工作流继续优化 —— 让 700+ KP 真正参与产出 ✅

**任务**：用户追问“700 多个 KP 怎么更好地应用于备课产出”，核心问题不是再补几个链接，而是把 `KP` 从“frontmatter 挂名资产”升级成“必须有产出落点的抽料来源”。

### 一、本次改进的核心判断

当前流程里虽然已经强调“读取范围内 KP”，但还缺一层更强的约束：

- `referenced_kps` 很容易只变成列名
- 备课 Agent 容易只真正使用 1-2 个 KP
- 例题、易错点、图片、推导并没有被强制回溯到具体 KP

所以这次改动的重点，不是“多读一些 KP”，而是“让每个被引用的 KP 都留下使用痕迹”。

### 二、已写入流程的规则

#### 1. 在 `Agent-10-备课-轻量` 中新增

- `KP 使用审计表`
- `KP 分层使用` 口径：
  - 主干 KP
  - 支撑 KP
  - 题型 KP
  - 易错 KP
  - 图景 KP
- 明确要求每个核心板块回答：
  - 定义来自哪个 KP
  - 原理 / 推导来自哪个 KP
  - 易错点来自哪个 KP
  - 例题链来自哪个 KP / 专题 / 题库
  - 图片来自哪个 KP / 专题 / 教材图

#### 2. 在 `模板-课件加工页` 中新增

- `2.4A KP 使用审计`
- `referenced_kps 大部分都有实际落点` 的覆盖核对项
- 验收清单中新增：
  - `KP 使用审计已填写`
  - `referenced_kps` 不是只挂名

#### 3. 在 `Agent标准作业流程（SOP）` 中新增

- Phase 0 自检增加：
  - 备课时是否真的把 KP 当抽料资产，而不是只把 `referenced_kps` 当字段
- 让 SOP 明确提出：
  - `KP` 需要能说清对定义、机制、例题、易错点、图景分别贡献了什么

### 三、这套规则的真正作用

如果后续 agent 按这版流程工作，`KP` 的用途就不再只是“知识库背景板”，而会更像五类可调用资产：

1. **主干 KP**
   - 决定一节课必须讲什么
2. **支撑 KP**
   - 补机制、推导、比较维度
3. **题型 KP**
   - 负责把知识真正落到例题链
4. **易错 KP**
   - 负责课堂上的纠偏价值
5. **图景 KP**
   - 负责图片、图像说明和可视化理解

### 四、当前结论

700 多个 `KP` 真正有价值的前提，不是“全部平铺到讲义里”，而是让它们在备课时形成：

- 分层调用
- 可回溯使用
- 与例题 / 易错点 / 图片绑定

这样数量优势才会真正转化为产出质量优势。

## 📌 2026-06-17 工作流补丁 —— 把 PPT 字体与公式验收写成硬约束 ✅
