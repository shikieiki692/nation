- 输出：迁移报告（字段变更清单 + 段落完整度对比）

#### 建议 2：新增 Agent 12 · 断链修复 Agent（构建/维护边界）
**问题**：Agent 9 能发现断链，Agent 5 能建知识点↔题目链接，但**中频断链（引用 5-7 次的目标）的批量创建**没有专门 Agent。状态摘要显示仍有大量中频断链未处理。
**方案**：
- 触发：kb-link-doctor / Agent 9 扫描后输出的断链清单
- 操作：按断链频率分层处理——高频（≥8）自动创建框架 + 中频（5-7）用户确认后创建 + 低频（1-4）记录到待办
- 前置：检查别名（如 `羟醛缩合` → `Aldol缩合`）避免误建
- 后置：更新所有引用文件的 wikilink

#### 建议 3：新增 Agent 13 · 多源对照 Agent（构建型）
**问题**：同一知识点（如「配合物」）可能有《普通化学原理》《无机化学》《Arrow Pushing》等多本教材的提炼笔记。当前各提炼笔记独立存在，**不同教材的讲法差异、侧重点、陷阱点没有被聚合到 KP 中**。
**方案**：
- 触发：用户指定某 KP，或 Agent 1 发现某章有多个源材料
- 操作：读取该 KP 关联的所有提炼笔记 → 做多源 diff → 提取差异点
- 输出位置：补充到 KP 的「七、常见比较与易混点」或「十二、教学视角」
- 价值：让 KP 的"易混点"和"教学视角"基于真实教材差异，而非 AI 编造

#### 建议 4：正式化 Agent 11 · 答疑 Agent（使用型）
**问题**：当前 [[Agent提示词-知识库使用]] 中 Agent 11 为"答疑（占位）"，无具体提示词。05-问答/ 目录已创建但为空。
**方案**：
- 定位：基于 KB 的**结构化答疑**（区别于 Agent 10 的备课输出）
- 输入：学生问题 + `target_kp[]`（可选，限定知识范围）
- 输出格式：
  ```
  问题拆解 → 涉及 KP 链接 → 分步解答 → 常见错误预警 → 延伸追问
  ```
- 特点：回答中每个结论必须附带 wikilink 到具体 KP；若问题跨越多个 KP，自动建议学习路径
