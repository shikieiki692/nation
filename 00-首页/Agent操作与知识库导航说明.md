# Agent操作与知识库导航说明

> 本文件解释 Claudian（AI Agent）在知识库中的操作方式，帮助你理解文件结构、跟踪进度、自主导航。
> 说明重点是“Agent 通常怎么读、怎么做、做完回写哪里”；若你只是日常找入口，优先看 [[首页]]。

---

## 一、Agent 做了什么？（操作速览）

当你发出指令后，Agent 会按照 **Phase 0-4 标准作业流程** 执行：

```
Phase 0: 加载上下文（默认读 [[状态摘要]] → [[活跃任务]] → [[Agent战略方向]]，跨模块大批量任务时读完整模式）
    ↓
Phase 1: 制定计划（确认目标、选定工具、预判风险）
    ↓
Phase 2: 执行操作（批量创建/修改文件、提取资料）
    ↓
Phase 3: 验证抽查（检查 3-5 个产出文件的质量）
    ↓
Phase 4: 归档收尾（记录工作日志、更新活跃任务、更新索引、同步状态摘要/战略方向）
```

**常用记录入口如下：**
- `00-首页/状态摘要.md` —— 会话快照与当前风险
- `00-首页/活跃任务.md` —— 执行聚合与健康检查
- `00-首页/Agent战略方向.md` —— 长期优先级与默认推进方向
- `00-首页/Agent入口文件关系说明.md` —— 入口顺序、职责边界与冲突处理
- `00-首页/工作日志/YYYY-MM-DD.md` —— 每次对话的过程记录与证据
- `00-首页/Agent最小执行协议.md` / `00-首页/Agent模块关系与收工同步清单.md` —— 默认执行路径与跨模块速查

## 三、Dataview 查询是什么？（动态表格）

你在很多页面底部看到的「关联真题」表格是 **Dataview 插件**自动生成的，不需要手动维护。

### 它的工作原理

```dataview
TABLE year, exam_stage, difficulty
FROM "04-题库/真题"
WHERE contains(knowledge_points, this.file.name)
```

**白话解释**：
- 扫描 `04-题库/真题/` 文件夹下的所有题目
- 检查每道题的 `knowledge_points`（知识点标签）
- 如果标签中包含「当前页面名称」，就把这道题列出来

### 你看到的效果

| 题目 | 年份 | 题型 | 难度 |
|------|:---:|:---|:---:|
| 题-035-7-过渡金属配合物推断 | 2021 | 推断题 | 5 |

> ✨ **这意味着**：每当你新建一道真题并正确填写 `knowledge_points`，所有相关 KP 页面会自动「长」出这条记录。

### 如何手动刷新

如果新增题目后表格没有更新：
1. 按 `Ctrl + P` 打开命令面板
2. 输入 `Dataview: Force refresh all views`
3. 回车

## 四点五、Agent 90 秒定位法

如果任务跨两个及以上模块，推荐 Agent 按下面的顺序定位：

```text
状态摘要
  ↓
活跃任务（先看健康检查，再看任务优先级）
  ↓
Agent模块关系与收工同步清单
  ↓
对应模板 / 对应模块真相源
```

例子：

| 用户说… | 先读什么 | 做完重点回写 |
|---|---|---|
| “把这套网课整理成下周可直接上课的材料” | 状态摘要 → 模块关系清单 → 方法论 → 备课待办 → 模板 | 工作日志 + 活跃任务 + 备课待办 + 必要时状态摘要 |
| “把这个专题做成学生可复习的材料” | 状态摘要 → 模块关系清单 → 专题页 / 课件 / 学生侧模板 | 工作日志 + 活跃任务 + 学生侧目录 / 阻塞状态 |
| “继续推进第四轮” | 状态摘要 → 活跃任务 → 战略方向 → 备课待办 | 工作日志 + 活跃任务 + 备课待办 + 必要时首页/状态摘要 |

> 入口链路先回答“现在什么情况、现在做什么”；模块关系清单再回答“这次改动会牵动哪几层、最后要回写哪几处”。
> 若只是人工浏览，不需要严格按这条链路逐页阅读。

## 五、Frontmatter 字段速查

每个文件顶部的 YAML 块（两个 `---` 之间）是文件的「身份证」。

### 真题文件常用字段

| 字段 | 示例值 | 用途 |
|------|--------|------|
| `type` | 题目 | 区分题目/知识点/索引 |
| `year` | 2021 | 按届筛选 |
| `module` | 无机和结构化学 | 按模块筛选 |
| `question_type` | 推断题 | 按题型筛选 |
| `difficulty` | 1~5 | 按难度筛选 |
| `teaching_level` | 入门/巩固/拓展/竞赛拔高 | 按教学层次筛选 |
| `knowledge_points` | [配合物, 晶体场理论] | **Dataview 关联查询的关键** |
| `tags` | [化竞, 真题, 35届] | 快速标签检索 |

### 知识点文件常用字段

| 字段 | 示例值 | 用途 |
|------|--------|------|
| `type` | 知识点 | 区分类型 |
| `status` | 骨架/初稿/已填充 | 填充进度跟踪 |
| `syllabus_code` | [12] | 对应考纲条目编号（单条单个值，多条用数组）|
| `importance` | 1~5 | 知识点重要程度 |

## 六、常见问题（FAQ）

**Q1: Agent 修改了文件，但我没看到变化？**
> A: Obsidian 有时不会实时刷新外部修改。按 `Ctrl + R` 刷新当前页面，或切换一下笔记再切回来。

**Q2: Dataview 表格显示「0 条结果」？**
> A: 检查两点：(1) Dataview 插件是否已启用；(2) 当前页面的文件名是否精确匹配题目 `knowledge_points` 中的值。注意大小写和全角半角。

**Q3: 我想自己添加一道真题，应该复制哪个模板？**
> A: 复制 `11-模板/模板-真题导入检查清单.md` 中的完整示例，或参照任意已有的 `题-035-xxx.md` 文件。

**Q4: Agent 提到的 "follow-up" 是什么意思？**
> A: 这是 Agent 在本次操作完成后建议的「下一步行动」。现在短期 follow-up 默认落到 `00-首页/活跃任务/` 任务卡，并由 `[[活跃任务]]` 聚合展示；工作日志只保留过程与证据；这些建议不会自动执行，除非你下次明确授权，或你说“继续推进”让 Agent 按优先级自主处理。

**Q5: "🔴 未建 / 🟡 初稿 / ✅ 已建" 这些标记什么意思？**
> A: 这是题目/知识点的工作状态图例：
> - 🔴 未建：文件尚未创建
> - 🟡 初稿：文件已创建但内容不完整
> - ✅ 已建：文件已完成并验证

## 七、如何与 Agent 高效协作

### ✅ 高效指令示例

| 低效指令 ❌ | 高效指令 ✅ | 原因 |
|-----------|-----------|------|
| "帮我整理一下真题" | "将 35届初赛第3题拆分为独立文件，按模板格式" | 明确届号、题号、格式要求 |
| "检查有没有错误" | "抽查题-035-7的frontmatter完整性，重点检查knowledge_points是否有broken link" | 明确范围、明确检查项 |
| "继续" | "继续创建35届剩余未建题目，从题-035-3-2开始" | 明确起点和终点 |

### 🚫 Agent 的红线（不会做的事）

1. **不会整体重写已填充 KP**：如果文件已存在且 status = "已填充"，Agent 默认只追加/修正，不整体重写。例外：明确触发的 Agent 7（KP 优化）或模板迁移任务，可在保留内容的前提下升级 frontmatter/段落结构
2. **不会删除文件**：如需清理，会建议你先确认
3. **不会修改 `.obsidian/` 配置文件夹**：除非明确授权

## 八、Vault 本地脚本工具速查

> 以下脚本存放在 `11-模板/scripts/` 下，是 Agent 可调用的批量化工具。
> 使用前确认依赖已安装：`pip install python-docx pyyaml`

| 脚本 | 用途 | 调用方式 |
|:---|:---|:---|
| `build-all-handout-docx.py` | 将 `04-课件/学生讲义/` 下全部 .md 讲义一键转换为格式统一的 .docx（A4 页面、微软雅黑 + Times New Roman、表格/公式/代码块均处理） | `python build-all-handout-docx.py` |
| `docx_utils.py` | 共享样式库（被上述脚本调用），也可被其他手写 build 脚本 `from docx_utils import *` 复用 | `from docx_utils import base_doc, title_block, heading, ...` |
| `render_docx_windows.py` | docx → PDF → PNG 渲染（依赖 LibreOffice + pypdfium2），用于生成讲义预览图 | `python render_docx_windows.py input.docx` |

### build-all-handout-docx.py 常用参数

```bash
python build-all-handout-docx.py                    # 全部转换（34 份）
python build-all-handout-docx.py --file 滴定分析    # 单份讲义
python build-all-handout-docx.py --dry-run          # 预览清单
python build-all-handout-docx.py --verbose          # 调试日志
```

输出目录：`06-学生侧材料/讲义/`
