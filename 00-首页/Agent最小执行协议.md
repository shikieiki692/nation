# Agent最小执行协议

> **用途**：给 Agent 一份日常默认执行的最小规则集。
> 目标不是覆盖全部细节，而是让 Agent 在**更听话、更省 token**的前提下，稳定完成“路由 → 执行 → 收工同步”闭环。

> **默认原则**：普通任务先读本页，按本页 + 对应轻量提示词执行；只有复杂、高风险、跨模块或多轮失败任务，才升级到 [[Agent标准作业流程]] 和长版提示词。

**相关入口**

- 入口职责边界：[[Agent入口文件关系说明]]
- 执行聚合与健康检查：[[活跃任务]]
- 跨模块速查：[[Agent模块关系与收工同步清单]]
- `.base` / 数据库工具怎么配合：[[Agent数据库使用指南]]
- 完整规则书：[[Agent标准作业流程]]

## 一、默认读取顺序（普通任务）

> **Session.INIT 时一次性加载**：`[[状态摘要]]` → `[[活跃任务]]` → `[[Agent战略方向]]` 三文件在会话初始化时一次性读取并缓存，后续任务不再逐次重读；仅当任务跨模块时按需补读 `[[Agent模块关系与收工同步清单]]`。

```text
用户指令
  ↓
Session.INIT 已加载：[[状态摘要]] + [[活跃任务]] + [[Agent战略方向]]
  ↓
若跨两个及以上模块 → 按需补读 [[Agent模块关系与收工同步清单]]
  ↓
对应轻量提示词（11-模板/Agent-*-轻量）
  ↓
对应模板 / 模块真相源
  ↓
开工
```

### 真相源优先级

| 你想知道什么 | 优先读取 |
|---|---|
| 现在库里什么状态 | [[状态摘要]] |
| 眼下还有什么没做 / 任务卡是否缺字段 | [[活跃任务]] |
| 用户没指定任务时该往哪推 | [[Agent战略方向]] |
| 这次任务属于哪层、做完回写哪里 | [[Agent模块关系与收工同步清单]] |
| 这个 Agent 该怎么做 | 对应 `Agent-*-轻量` |
| 产出长什么样 | 对应 `[[11-模板/模板]]` 或相关具体模板文件 |
| 某模块具体放哪、谁是上下游 | 该模块真相源文件 |

> **健康检查优先**：如果 `[[活跃任务]]` 的健康检查提示缺 `status / priority / area`、证据字段为空、或 `completed` 漏日期，先修任务卡，再判断任务优先级。

> **边界提醒**：本页只解决“普通任务先读什么、何时升级”。如果你已经明确要用 `.base`、数据库页、工作日志拆分结构来筛选或导航，细节统一去看 [[Agent数据库使用指南]]，不要在本页继续展开工具层说明。

> **跨模块**：若任务类型涉及 ≥2 个模块（如"备课"同时修改了知识点），按"收工必须同步"列中**交集最大**的那个执行。

### 提示词补充策略

- **默认先用轻量提示词**（上表所列）
- **以下情况补读长版**（[[Agent提示词-知识库构建]] / 维护 / 使用）：
  1. 任务有多个子模式，轻量版不足以判断边界
  2. 涉及批量操作、迁移、合并、弃用
  3. 涉及 Agent 1A / 1B 等特殊子流程
  4. 前一次尝试已失败，怀疑遗漏细则
  5. 需要查某个字段、表格、规则的精确定义

### 完整 SOP 升级条件

满足以下任一，从本页升级到 [[Agent标准作业流程]]：
1. 跨 ≥2 个模块，且仍不确定上下游和收工回写
2. 会修改 SOP、模板、任务卡规则、系统入口文件
3. 批量改动文件较多，或涉及迁移/合并/弃用
4. 任务存在较高误改风险
5. 已经出现一次明显跑偏、漏同步或规则冲突

## 六、工作中遇到问题时先查哪里

```text
遇到问题
  ↓
先查 [[经验沉淀]] 有没有同类坑
  ↓
再查 [[工作日志]] 最近是否做过类似任务
  ↓
若仍不清楚 → 升级读长版提示词
  ↓
若仍高风险 / 会影响系统层 → 升级完整 SOP
```

### 问题类型 → 默认动作

| 问题类型 | 默认动作 |

## 七、收工 20 秒清单

| 发生了什么 | 必须同步哪里 |
|---|---|
| 只是完成了过程，没有改变结论 | `[[工作日志]]` |
| 某个待办状态变化了 | 对应 `00-首页/活跃任务/任务卡-*.md` |
| 生成或修改了 `04-课件/` 文件 | `[[工作日志]]` + `[[活跃任务]]` + `[[备课思路/备课与教学思路待办]]` |
| 更新了专题级课件进度判断 | `[[04-课件/专题产出总览看板]]` + `[[工作日志]]` |
| 新增了 `docx` 级大纲 / 讲义导出或 render 预览 | `[[工作日志]]` + 必要时 `[[04-课件/专题产出总览看板]]` |
| 阶段 / 阻塞 / 下一步变了 | `[[状态摘要]]` |
| 长期优先级 / 待用户决策项变了 | `[[Agent战略方向]]` |
| 出现可复用的新坑 / 新规避法 | `[[经验沉淀]]` |
| 获得真实课堂反馈 | `[[模板-复盘记录]]` 对应产物 |

### 经验沉淀判定

只有符合以下任一条件，才值得写入 [[经验沉淀]]：

1. 同类坑以后大概率还会再踩
2. 这个坑不是某个单文件特例，而是工作流级问题
3. 这条规避法能直接节省后续 Agent 的时间或返工

## 八、什么时候说明工作流是完整的

一条完整的工作链至少应能回答下面 8 个问题：

1. 用户这句话属于哪个意图？
2. 我该先读哪几个入口文件？
3. 我该用哪个轻量提示词？
4. 我该读哪个模板或模块真相源？
5. 我遇到问题先去哪查？
6. 我做完后哪些记录文件要更新？
7. 哪些结论该进活跃任务，哪些该进经验沉淀？
8. 哪些场景必须升级到长版提示词或完整 SOP？

> 如果这 8 个问题里有任意一个答不上来，就不要硬做，先升级读取层级。
