# 图片索引 · 全库图片资产入口

> **本页定位**：这是当前 vault 的权威图片导航页。Agent / 人在备课、补 KP、写讲义时，先来这里决定“去哪找图、找到后怎么落地”。
> **配套规则**：与 [[规则-配图来源优先级]]、[[11-模板/MD讲义格式参考（原子结构样板）]]、[[11-模板/LaTeX PDF管线使用手册]] 一起使用。
> **维护方式**：大目录由 DataviewJS 自动列；主题分类、精标表、OCR 噪声黑名单与检索 hint 由 Agent 增量维护；路径一致性可用 `python 11-模板/scripts/audit_image_index.py` 定期核验。

> **页面边界**：本页只负责“图去哪里找、找到后怎么沉淀”，不负责决定“这节课该不该配图、课堂顺序怎么排、讲义具体放哪一段”。
> 配图是否必要与课堂顺序：优先回对应 `04-课件/` 页。
> 学生讲义的最终图片落地：统一回 vault 根目录 `media/`。

## 当前权威口径（先看这里）

1. 本页是默认第一入口，先复用已有图，再重新找图。
2. `图片索引` 负责**发现与导航**，不是学生讲义的最终存储目录。
3. 学生讲义最终使用的图片，一律落到 vault 根目录 `media/`，讲义正文统一写 `![[media/...]]`。
4. `mineru/`、教材 OCR、真题原图、真题解析图是**来源仓**；进入讲义前先复制到 `media/`。
5. 新图片默认使用稳定英文文件名，不再新增 `Pasted image ...`、中文临时名、同图多份改名副本。
6. 图注只写图中对象和教学用途，控制在 30 汉字左右；长说明、来源说明回正文，不写在图注里。
7. 当前 [[原子结构-超级充实版（自学完整）.md]] 的既有图片引用已视为修复完成，本页规范主要约束后续新稿与重构稿。

## 什么情况下需要更新本页

1. 找到了一张以后大概率还会复用的图
2. 某主题第二次仍要重复检索，说明需要补精标或 hint
3. 新识别出一类 OCR 噪声图，值得进黑名单
4. 某张图已经形成图注笔记，值得建立“主题→图”快速通道

## 什么情况下不必更新本页

1. 一次性截图，只服务当次临时说明
2. 图像只是过程草稿，不会进入正式讲义或 KP
3. 已有精标条目和图注笔记，本次没有新增信息

## 📂 已有图片仓（按目录）

> 自动扫描全库 .jpg/.jpeg/.png/.gif/.webp/.svg 文件，按目录聚合。空目录不显示。

```dataviewjs
// 扫描全库图片，按所在目录聚合
const exts = ["jpg", "jpeg", "png", "gif", "webp", "svg"];
const allImages = app.vault.getFiles().filter(f =>
  exts.includes(f.extension.toLowerCase())
);

const byDir = {};
for (const f of allImages) {
  const dir = f.parent ? f.parent.path : "/";
  if (!byDir[dir]) byDir[dir] = [];
  byDir[dir].push(f);
}

// 按图片数量倒序排
const sortedDirs = Object.keys(byDir).sort((a, b) => byDir[b].length - byDir[a].length);

if (sortedDirs.length === 0) {
  dv.paragraph("（暂无图片）");
} else {
  dv.table(
    ["目录", "图片数", "示例文件"],
    sortedDirs.slice(0, 30).map(dir => [
      dir,
      byDir[dir].length,
      byDir[dir].slice(0, 2).map(f => f.name).join(", ") + (byDir[dir].length > 2 ? " ..." : "")
    ])
  );
}
```

### 检索字段说明
- **主题**：化学概念/反应/题型/结构的标准名（尽量与 KP 标题写法一致）
- **路径**：`mineru/<完整路径>/<hash>.jpg` 完整相对路径（Agent 拷贝即可用）
- **关联 KP / 真题**：wikilink + 真题届数 + 题号（便于反向查找）
- **上下文摘要**：图前后 1-2 句的题面/章节标题（便于二次确认）
- **类型**：晶胞 / 机理 / 谱图 / 装置 / 公式 / 流程 / 题面
- **置信度**：🟢 已视觉验证 / 🟡 仅上下文判断 / 🔴 仅推测

## 🔧 复用规范

### 1. 最终讲义引用格式
- 学生讲义最终稿只使用：`![[media/xxx.png]]`
- 允许设定显示宽度：`![[media/xxx.png|300]]`
- 不再把 `mineru/...`、`06-外部资料导入/...` 直接挂进最终学生讲义
- `Pasted image XXX.png` 只作为历史兼容，不再新增

### 2. 图片落地原则

| 场景 | 规则 |
|:---|:---|
| 还在找图阶段 | 可以在笔记中暂时记录来源路径 |
| 确认进入学生讲义 | 先复制到根目录 `media/` |
| 进入讲义正文 | 统一写 `![[media/稳定英文名.ext]]` |
| 复用价值高 | 回写本页精标表 / 图注笔记 |

### 3. 何时反哺到 KP

| 触发时机 | 操作 |
|:---|:---|
| 课件成形后 | 挑 3-5 张关键图，嵌入对应 KP，更新 KP frontmatter `has_images: true` |
| KP 优化时（Agent 7）| 发现现有 mineru 图可用 → 嵌入 KP，更新字段 |
| 健康检查（Agent 9）| 扫描 `images_priority: high` 但 `has_images: false` 的 KP，列入待补清单 |
| 图注笔记创建后 | 更新对应 KP 的 `key_images: ["[[10-附件/图注/xxx]]"]`，建立知识点→图的快速通道 |

### 4. 优先级判断

| KP 类型 | images_priority | 是否必须补图 |
|:---|:---:|:---:|
| 几何/空间结构 | high | 强烈推荐 |
| 反应机理 | high | 强烈推荐 |
| 谱图典型 | high | 推荐 |
| 实验装置 | medium | 视情况 |
| 概念定义 | low | 不必 |
| 计算公式 | low | 不必 |

## 📊 待补图清单（DataviewJS 自动）

> 自动列出 `images_priority: high` 且 `has_images: false` 的 KP，备课/填充时优先处理。
> 若此表为空，说明当前高优待补已清零；之后继续按“备课驱动、命中即补”的方式维护即可。

```dataview
TABLE
  subject AS 模块,
  importance AS 重要度,
  status AS 当前状态
FROM "03-知识点"
WHERE type = "知识点"
  AND images_priority = "high"
  AND has_images != true
SORT importance DESC
```
