# 自动汇总 · 说明

> 本目录是 `00-首页/` 系统页的**中间汇总层**。
> 只放“可自动聚合的事实结果”，不放长期判断、优先级解释和策略结论。

## 设计目的

- 把“真相源”和“展示页”拆开
- 让 `[[状态摘要]]`、`[[活跃任务]]`、`[[首页]]` 只负责读取结果与做少量人工判断
- 为后续 Dataview / DataviewJS / 脚本化汇总预留稳定出口

## 和其他入口的关系

| 层级 | 负责什么 | 典型页面 |
|---|---|---|
| 真相源层 | 字段、任务卡、日志、源文件本身 | `00-首页/活跃任务/`、`03-知识点/`、`00-首页/工作日志/` |
| 自动汇总层 | 把可计算事实聚合成稳定中间结果 | 本目录各页 |
| 系统入口层 | 读取自动事实，再补少量人工判断 | [[状态摘要]]、[[活跃任务]]、[[首页]] |
| 决策层 | 给出方向、取舍、阻塞解释 | [[Agent战略方向]] |

> 一句话记忆：真相源负责“数据从哪来”，自动汇总负责“数据怎么算”，系统入口负责“现在给人看什么”，战略层负责“为什么现在优先这个”。

## 使用边界

- 本目录页面优先放：
  - 版本聚合
  - 任务卡健康检查
  - 库存统计
  - 审计摘要索引
- 本目录页面不负责：
  - 阶段判断
  - 当前重心
  - 阻塞解释
  - 优先级取舍

## 维护原则

1. 真相源变了，先修真相源，再看自动汇总页
2. 自动汇总页错了，优先修聚合逻辑，不要先改入口页文案
3. 入口页只做展示与跳转，不把自动逻辑复制回正文
4. `.base` / Dataview 视图主要负责“筛选和浏览”，不要让它们承担阶段判断或任务优先级解释
