# 系统页角色索引

> 本页用来验证 `00-首页` 系统页的 `role` 标注是否足够稳定、是否便于分层阅读与后续自动化。
> 若只是判断当前先做什么，仍先走 `[[状态摘要]] → [[活跃任务]] → [[Agent战略方向]]`；本页只负责“系统页按角色怎么分层看”。

> 角色口径定义见 [[Frontmatter角色词表]]。
> `type` 枚举现统一为 `系统 / 活跃任务卡 / 工作日志`，本页只索引 `type: 系统` 的页面。
> **本轮补充（2026-06-15）**：除了主链路页，也开始明确收纳“任务拆分页 / 资源索引页 / 日志索引页 / 方法论页”，避免这些高频辅助页长期游离在角色体系之外。

---

## 一、按角色浏览

```dataview
TABLE WITHOUT ID
  role AS 角色,
  file.link AS 页面,
  type AS 类型,
  updated AS 更新日期
FROM "00-首页"
WHERE type = "系统"
SORT role ASC, file.name ASC
```

## 三、漏标巡检

> 下面若出现结果，说明该页还没有补 `role`，或类型字段不在当前约定范围内。

```dataview
TABLE WITHOUT ID
  file.link AS 页面,
  type AS 类型,
  updated AS 更新日期
FROM "00-首页"
WHERE type = "系统" AND !role
SORT file.name ASC
```

## 四、使用建议

1. 想找“现在该读什么”，不要从本页起步，回主链路。
2. 想检查某页应该归哪一类，先看 [[Frontmatter角色词表]]，再回本页看实际挂载效果。
3. 想做 Dataview / Bases / 自动巡检时，优先复用这里已经收口的 `role` 口径。
4. 若本页“漏标巡检”出现结果，优先判断是该页真的缺 `role`，还是词表尚未吸收新的稳定辅助页类型。
