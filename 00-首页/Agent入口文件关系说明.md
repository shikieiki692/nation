# Agent入口文件关系说明

> **用途**：说明 `[[状态摘要]]`、`[[活跃任务]]`、`[[Agent战略方向]]` 三份入口文件各自负责什么、应该按什么顺序读取、出现冲突时如何处理。
> **字段规范补充**：若需统一 `type / role` 等 frontmatter 取值，见 [[Frontmatter角色词表]]；若想检查实际挂载效果与漏标页，见 [[系统页角色索引]]。

**相关入口**

- 会话快照：[[状态摘要]]
- 执行聚合：[[活跃任务]]
- 长期方向：[[Agent战略方向]]
- 跨模块回写：[[Agent模块关系与收工同步清单]]

## 一、默认读取顺序

```text
[[状态摘要]]
  ↓
[[活跃任务]]
  ↓
[[Agent战略方向]]
  ↓
若任务跨模块或收工回写不明确，再看 [[Agent模块关系与收工同步清单]]
```

## 二、三份入口文件的职责分工

| 文件 | 层级 | 核心问题 | 不负责什么 |
|:---|:---|:---|:---|
| [[状态摘要]] | 会话快照层 | 现在处于什么状态、最近做了什么、接下来先看哪里 | 不维护完整短期任务明细 |
| [[活跃任务]] | 执行聚合层 | 现在有哪些任务在做、哪些已完成、哪些被阻塞、任务卡是否健康 | 不负责长期优先级判断 |
| [[Agent战略方向]] | 长期方向层 | 知识库接下来往哪走、哪些方向优先、有哪些待用户决策事项 | 不承载短周期任务清单 |

## 三、推荐使用方式

1. **用户问“现在什么状态 / 下一步做什么”**
   先读 `[[状态摘要]]`，再到 `[[活跃任务]]` 核对执行层真相。

2. **用户说“继续推进 / 开始工作”**
   先看 `[[活跃任务]]` 的健康检查和 P1/P2/P3，再用 `[[Agent战略方向]]` 判断默认推进方向。

3. **用户说“为什么现在优先做这个”**
   先用 `[[活跃任务]]` 说明执行层依据，再用 `[[Agent战略方向]]` 说明战略层依据。

4. **任务跨模块、收工时不确定回写哪里**
   打开 `[[Agent模块关系与收工同步清单]]`，按模块决定同步目标。

## 四、冲突处理规则

1. 若 `[[状态摘要]]` 与 `[[活跃任务]]` 冲突：
   以 `00-首页/活跃任务/` 任务卡聚合结果为准，先修任务卡或入口页健康检查暴露的问题。

2. 若 `[[活跃任务]]` 与 `[[Agent战略方向]]` 冲突：
   先确认该差异是“短期执行变化”还是“长期方向变化”。
   短期执行变化只修任务卡；长期方向变化再回写 `[[Agent战略方向]]`。

3. 若三者都与文件系统实际产出冲突：
   以真实文件和审计结果为准，再回写三层入口。

## 五、维护原则

- `[[状态摘要]]`：偏“读者视角”，负责快速进入上下文。
- `[[活跃任务]]`：偏“执行视角”，负责把任务卡聚合结果和健康检查暴露出来。
- `[[Agent战略方向]]`：偏“决策视角”，负责说明为什么当前优先做这类事。

> 一句话记忆：
> `状态摘要` 看全局，`活跃任务` 看执行，`Agent战略方向` 看方向。
