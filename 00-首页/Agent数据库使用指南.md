# Agent 数据库使用指南

> 本文件说明 `.base` 数据库、`[[活跃任务]]` 聚合页、`00-首页/活跃任务/` 任务卡目录和按天拆分的工作日志如何提升 Agent 工作效率，以及 Agent 在执行任务时应如何利用这些结构。
> 它属于**工具层说明页**：当你需要理解 `.base` / 任务卡 / 工作日志怎么配合时再来这里；日常任务默认仍先走 `[[状态摘要]] → [[活跃任务]] → [[Agent战略方向]]`，数据库页本身不承担阶段判断。

---

## 一、为什么数据库格式能提升 Agent 效率？

### 1.1 数据源集中化

| 场景 | 旧方式（大单文件） | 新方式（`.base` + 拆分日志） |
|:---|:---|:---|
| **读取工作日志** | 加载 600+ 行的单文件，大部分内容无关 | 只加载当天的 `YYYY-MM-DD.md`，按需读取 |
| **理解任务范围** | 从长文本中人工找 Follow-up | `[[活跃任务]]` + `.base` 过滤条件一目了然 |
| **更新日志** | 在单文件末尾追加，容易冲突 | 新建或修改当天文件，隔离性好 |
| **状态跟踪** | 无结构化视图 | Bases 表格实时聚合状态 |

### 1.2 对 Agent 的具体收益

1. **Phase 0 加载更快**：Agent 读取 `02-数据库/工作流.base` 可快速理解日志结构，无需先扫整个 `00-首页/`
2. **Phase 4 归档更规范**：每天一个文件，Agent 新建 `2026-05-09.md` 而不会污染历史记录
3. **上下文更精准**：用户说"继续昨天的工作"，Agent 直接读 `2026-05-08.md` 而非全文搜索
4. **Follow-up 可追溯**：`.base` 的筛选条件可作为辅助筛选器，但短期任务真相源仍以任务卡目录和 `[[活跃任务]]` 为准

## 二、数据库文件结构速查

### 2.1 文件位置

```
02-数据库/
├── 知识点.base      → 聚合 03-知识点/，按模块/状态/难度筛选
├── 真题.base        → 聚合 04-题库/真题/，按届次/模块/题型筛选
├── 工作流.base      → 聚合 00-首页/工作日志/，按日期排序
└── 外部资料.base    → 聚合 07-资料提炼/，按审核状态筛选
```

### 2.2 `.base` 文件语法（Agent 只读即可）

```yaml
filters:              # 全局筛选条件
  and:
    - file.inFolder("03-知识点")
    - type == "知识点"

properties:           # 定义显示的列
  file.name:
    displayName: 知识点
  status:
    displayName: 状态

views:                # 视图定义
  - type: table
    name: 知识点总览
    order:
      - file.name
      - status
    sort:
      - property: importance
        direction: DESC
```

**Agent 读取 `.base` 的关键字段**：
- `filters` → 理解"这个数据库关注哪些文件"
- `properties` → 理解"用户关心哪些元数据"
- `views[].filters` → 理解"用户预设了哪些常用筛选"

## 三、Agent 工作流程中的应用

### 3.1 Phase 0：加载上下文

**标准读取顺序**（含默认/完整模式）→ [[Agent最小执行协议#一、默认读取顺序（普通任务）]]。

**本层新增**：

```text
读完 状态摘要 → 活跃任务 → 战略方向 之后：
  【可选】扫一眼 02-数据库/*.base 了解当前关注范围
  读当天工作日志（如有）
```

> **示例**：打开 `知识点.base`，看到视图”待填充高优”的筛选条件是 `status != “已填充” AND importance >= 4`。这能帮助快速筛出候选范围；但是否真要补这些知识点，仍以 `活跃任务` 和当前备课主线为准。

**跨模块任务入口顺序** → [[Agent模块关系与收工同步清单#二、从用户任务反推读取路径]]。

### 3.2 Phase 2：执行操作

当 Agent 需要批量创建/修改文件时，可以参考 `.base` 的 `filters` 来限定范围：

```yaml
# 从 真题.base 读取
filters:
  and:
    - file.inFolder("04-题库/真题")
    - type == "题目"
```

→ Agent 知道这个数据库只关心 `04-题库/真题/` 下的 `type == "题目"` 文件，不会误触其他目录。

### 3.3 Phase 4：归档（工作日志 + 任务卡写入规范）

**旧规范**：在 `00-首页/工作日志.md` 末尾追加条目。

**新规范**：
1. 确定今天的日期（如 `2026-05-09`）
2. 检查 `00-首页/工作日志/2026-05-09.md` 是否存在
   - 存在 → 在文件末尾追加新条目
   - 不存在 → 新建文件，按模板写入
3. 同步 `00-首页/活跃任务/`
   - 新未完成项 → 新建一张任务卡
   - 已完成项 → 在对应任务卡中改为 `status: completed` 并补 `completed`
3. 新文件模板：

```markdown
# 工作日志 · 2026-05-09

> 本文件记录 2026-05-09 的知识库建设任务与里程碑。

## 📅 2026-05-09 <任务标题>

### 任务类型
（如：4. 题库拆分 Agent）

### 范围
- **源文件**：`...`
- **目标产出**：`...`

### 核心数据
（关键指标表格）

### 验证情况
- **抽查样本**：...
- **通过维度**：...

### Follow-up
#### 🔥 高优先级
- [ ] ...
```

### 3.4 跨天任务续作

当用户说"继续"或"把昨天的 follow-up 做完"时：
1. Agent 先读取 `[[活跃任务]]`
2. 若需要上下文，再读取 `工作流.base` 或按 `file.mtime` 排序找到最近的工作日志
3. 打开对应日期的 `.md` 文件查看过程与证据
4. 按优先级逐一执行

## 四、工作日志拆分后的索引维护

### 4.1 原 `工作日志.md` 已变为索引

`00-首页/工作日志.md` 现在是一个**静态索引页**，包含：
- 按日期排序的日志列表（表格）
- `![[工作流.base]]` 嵌入视图
- Agent 速查说明

**Agent 不应再向此文件追加内容**。

### 4.2 状态摘要与活跃任务的自动聚合

`00-首页/状态摘要.md` 已更新：
- **Follow-up 查询**：`FROM "00-首页/活跃任务"`，按任务卡 `status / priority` 聚合
- **最近任务查询**：DataviewJS 扫描 `00-首页/工作日志/` 下所有文件，提取 `## 📅` 标题
- **定性状态**：仍由 Agent 在 Phase 4 同步维护

这意味着：
- 新增一天的日志 → 状态摘要自动显示最近任务
- 完成一项活跃任务 → 状态摘要自动更新待办看板
- `00-首页/活跃任务/` 任务卡成为短期执行真相源，`[[活跃任务]]` 变为聚合入口
- 但 `当前重心 / 阻塞 / 下一步` 仍需 Agent 在 Phase 4 手工同步
- 若任务跨模块，建议在收工前再扫一眼 `[[Agent模块关系与收工同步清单]]` 的“收工同步矩阵”，避免漏掉 `备课与教学思路待办`、`Agent战略方向` 等非自动聚合文件

---

### 4.3 活跃任务与工作日志的分工

| 文件 | 角色 | 写什么 | 不写什么 |
|:---|:---|:---|:---|
| `00-首页/活跃任务/` + `[[活跃任务]]` | 任务卡真相源 + 聚合看板 | 当前未完成事项、优先级、近期完成 | 长篇过程记录 |
| `[[工作日志]]` | 证据与归档 | 做了什么、为什么、验证结果、证据链接 | 完整短期任务主清单 |
| `[[状态摘要]]` | AI 启动快照 | 重心、阻塞、下一步、自动聚合看板 | 细碎执行过程 |

## 五、数据库 vs Dataview 的分工

| 需求 | 首选工具 | 原因 |
|:---|:---|:---|
| 快速筛选、排序、编辑属性 | **`.base`（Bases）** | 交互式表格，直接 inline 编辑 |
| 复杂聚合（GROUP BY、FLATTEN） | **`.md`（Dataview）** | 支持 `GROUP BY`、`average()`、`DataviewJS` |
| 跨文件夹关联查询 | **`.md`（Dataview）** | `FROM` 支持多路径，`contains()` 灵活 |
| 日常浏览和教学备课 | **`.base`（Bases）** | 视觉更清晰，响应更快 |
| Agent 自动解析任务范围 | **`.base`（Bases）** | YAML 结构化，比 Dataview 查询更易解析 |

**建议**：Agent 执行任务时以 **系统入口页判断当前方向**，以 **`.base` 为筛选器**，再以 **Dataview `.md` 为深度分析工具**。

## 六、快速参考：Agent 常用数据库路径

| 目的 | 文件路径 | 读取内容 |
|:---|:---|:---|
| 了解今天该干什么 | `00-首页/活跃任务.md` | 当前未完成任务聚合、优先级与快速入口 |
| 结构化查看任务卡 | `02-数据库/活跃任务.base` | 表格/卡片方式筛选任务状态、优先级与证据 |
| 快速判断上下游模块与收工回写 | `00-首页/Agent模块关系与收工同步清单.md` | 模块关系、任务→读取路径、收工同步矩阵 |
| 查看待填充知识点 | `02-数据库/知识点.base` → 视图"待填充高优" | `status != "已填充" AND importance >= 4` |
| 查看真题数据质量 | `02-数据库/真题.base` → 视图"高难度拔高题" | `difficulty >= 4` |
| 追踪提炼进度 | `02-数据库/外部资料.base` → 视图"进行中" | `review_status == "进行中"` |
| 读取最新任务详情 | `00-首页/工作日志/2026-05-09.md`（或最新日期） | H2 标题、过程记录和证据链接 |
