## 使用规则

1. 新任务优先新建一张任务卡到 `00-首页/活跃任务/`
2. 每张任务卡必须维护 frontmatter，至少包含：`status`、`priority`、`area`、`source_notes`、`related_notes`、`evidence`
3. 任务完成后，不删除卡片；改为 `status: completed` 并补 `completed`
4. 任务卡记录"还有什么没做"；任务经过与验证细节写入 `[[工作日志]]`
5. `[[状态摘要]]` 和数据库看板只从任务卡聚合，不再从日志正文扫描短期 follow-up
6. 若某任务已被审计结论吸收（如复用已有专题页、接受历史命名、整体闭环已覆盖），应直接关卡，不必再补冗余产物
7. 若某任务明确等待用户决策、实体打印验证、真实课堂反馈或外部资料到位，应改为 `blocked`，不要长期停留在 `active`
8. 若任务属于资料导入 / 资料提炼 / 网课提炼，关闭前必须确认相关提炼页已补“备课抽料资产标记”；否则只能记为“已入库待接入”，不应直接 `completed`

## 状态快照

> 让 Agent 一眼看清"现在到底有多少还在做、多少已完成、多少正在等外部条件"，避免 `blocked` 只写进 frontmatter 却在入口层隐身。

```dataview
TABLE
  length(rows) as 总数,
  length(filter(rows, (r) => r.status = "active")) as Active,
  length(filter(rows, (r) => r.status = "blocked")) as Blocked,
  length(filter(rows, (r) => r.status = "completed")) as Completed,
  length(filter(rows, (r) => r.status = "active" AND r.priority = "P1")) as "P1 Active"
FROM "00-首页/活跃任务"
WHERE type = "活跃任务卡"
```

## 健康检查

> 用于快速发现 frontmatter 漏填、完成态漏日期、以及证据字段为空的任务卡，避免入口页统计失真。

```dataviewjs
const pages = dv.pages('"00-首页/活跃任务"')
  .where(p => p.type === "活跃任务卡");

const issues = [];

for (const p of pages) {
  const missing = [];

  if (!p.status) missing.push("缺 status");
  if (!p.priority) missing.push("缺 priority");
  if (!p.area) missing.push("缺 area");
  if (!p.source_notes || (Array.isArray(p.source_notes) && p.source_notes.length === 0)) missing.push("缺 source_notes");
  if (!p.related_notes || (Array.isArray(p.related_notes) && p.related_notes.length === 0)) missing.push("缺 related_notes");
  if (!p.evidence || (Array.isArray(p.evidence) && p.evidence.length === 0)) missing.push("缺 evidence");
  if (p.status === "completed" && !p.completed) missing.push("completed 缺完成日期");

  if (missing.length > 0) {
    issues.push([p.file.link, missing.join("；"), p.updated ?? "-"]);
  }
}

if (issues.length === 0) {
  dv.paragraph("✅ 当前任务卡 frontmatter 关键字段完整，未发现 completed 漏日期问题。");
} else {
  dv.table(["任务卡", "异常", "updated"], issues);
}
```

## 状态维护速查

| 状态 | 何时使用 | 必填要求 |
|:---|:---|:---|
| `active` | 仍需继续推进，且当前不依赖外部决策/资料/反馈 | `status`、`priority`、`area`、`source_notes`、`related_notes`、`evidence` |
| `blocked` | 明确等待用户决策、打印验证、课堂反馈、外部资料 | 保留原有证据，并在正文写清阻塞原因 |
| `completed` | 已有明确产出或审计结论吸收，不再继续执行 | 除基础字段外，**必须补 `completed: YYYY-MM-DD`** |

## 任务面板

### 🔴 P1 高优

```dataview
TABLE WITHOUT ID
  file.link as 任务,
  area as 领域,
  join(related_notes, " · ") as 关联,
  join(evidence, " · ") as 证据,
  updated as 更新
FROM "00-首页/活跃任务"
WHERE type = "活跃任务卡"
  AND status = "active"
  AND priority = "P1"
```
