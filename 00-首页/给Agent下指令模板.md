# 给Agent下指令模板

> **用途**：帮你更稳定地触发正确工作流，减少 Agent 误判、减少补充确认、减少 token 浪费。
> 你不需要每次都写很长，只要把关键字段说清楚即可。
> 本页已合并“详细模板 + 超短口令”，日常直接从这里复制即可。

---

## 零、超短口令版

> 适合你已经知道自己要什么，只想快速触发正确工作流的时候直接发。

### 万能骨架

```text
做【任务】，范围【对象】，产出【结果】，不要【越界项】，做完同步【收尾项】。
```

例子：

```text
做第四轮备课，范围是溶液与相图，产出备课大纲，不要讲义，做完同步工作日志和备课待办。
```

### 高频口令

- `提炼这章，只做考纲内，产出资料提炼笔记，做完写日志。`
- `整理这套网课，产出教学逻辑提炼，不直接回填 KP。`
- `补全这个 KP，重点补教学视角和例题，不改别的文件。`
- `审一遍这批 KP，只修 frontmatter 低风险问题。`
- `扫一遍这个目录的断链 / 空壳 / 字段问题，先出报告。`
- `检查任务卡状态，把已完成但还 active 的清掉。`
- `为这个专题做第一轮基础班备课大纲，不做讲义。`
- `继续推进第四轮，先做下一个未完成专题的备课大纲。`
- `优化提示词，目标是更省 token，但别变松。`
- `把这套规则接到 SOP、导航和状态入口里。`

### 风险控制口令

- `先只出报告，别直接改。`
- `先修低风险问题，高风险列出来。`
- `先做最值钱的 20%，剩下挂 follow-up。`
- `先别批量改，先给我看计划。`
- `不要动 KP 正文，只动流程文档。`
- `不要生成讲义，先停在备课大纲。`

### 收尾口令

- `做完同步工作日志。`
- `做完同步活跃任务。`
- `做完同步状态摘要和战略方向。`
- `做完把新坑沉淀进经验沉淀。`
- `做完把备课待办一起更新。`

## 一、最短通用模板

```text
帮我做【任务类型】，
范围是【文件 / 模块 / 主题】，
产出要【目标产物】，
限制是【不做什么 / 做到什么程度 / 是否同步收尾】。
```

### 四个最关键的信息

| 你最好说清楚什么 | 例子 |
|---|---|
| **任务类型** | 提炼 / 填充 / 审计 / 备课 / 优化 / 建模板 |
| **范围** | 一本书、一章、一个专题、一批 KP、某个目录 |
| **目标产物** | 资料提炼笔记 / KP / 审计报告 / 备课大纲 / 讲义 / 工作流文档 |
| **边界** | 只出报告不修改 / 不动 KP / 不做学生讲义 / 做完同步日志和任务卡 |

## 二、推荐句式

### 1. 构建类

```text
把【资料名 / 章节】提炼进知识库，
范围只做【考纲内 / 指定主题】，
产出【资料提炼笔记 / 教学逻辑提炼 / KP】，
做完同步【工作日志 / 活跃任务】。
```

例子：

- 把《普通化学原理》第 3 章提炼进知识库，只做考纲内内容，产出资料提炼笔记，做完写工作日志。
- 把这套网课的酸碱理论部分整理成教学逻辑提炼，不直接回填 KP，做完如果发现缺口再挂活跃任务。

### 2. KP 填充 / 优化类

```text
补全 / 优化【某几个 KP】，
重点做【教学视角 / 例题 / 易错点 / frontmatter】，
不要改【其他模块 / 文件名 / 链接结构】，
做完给我一个简短报告。
```

例子：

- 补全 [[缓冲溶液]] 和 [[酸碱滴定曲线]] 的教学视角和例题，不改别的文件，做完汇报缺了什么。
- 审一遍 03-知识点/分析化学 下这批 KP 的 frontmatter，只修低风险问题，高风险先列出来。

### 3. 审计 / 维护类

```text
检查【目录 / 模块 / 文件集合】的【某类问题】，
先【只出报告 / 先修低风险问题】，
不要做【批量重命名 / 结构重构】，
最后同步【工作日志 / 活跃任务 / 状态摘要（如需要）】。
```

例子：

- 审计 04-专题与题型/专题 里的空壳专题页，先只出报告，不要直接大改。
- 检查这批课件的 related-lessons 字段，只修格式问题，不改正文。

### 4. 备课 / 讲义类

```text
为【专题 / KP / 轮次】准备【班型 + 时长】的【备课大纲 / 课堂讲义 / 复习课 / 习题课】，
重点讲【侧重点】，
暂时不做【专题页升级 / 学生侧材料 / PPT】，
做完同步【备课待办 + 工作日志 + 活跃任务】。
```

例子：

- 为 [[专题-酸碱理论]] 准备第一轮基础班 90 分钟备课大纲，重点讲认知台阶和例题穿插，暂不做学生讲义。
- 为第三轮有机“羰基化学与缩合反应”准备提高班讲义，重点讲机理和易错点，不做 PPT。

### 5. 系统 / 工作流类

```text
优化【SOP / 模板 / 导航 / 提示词】，
目标是【更省 token / 更少漏同步 / 更好路由】，
涉及到的入口文件都帮我同步，
最后写工作日志并说明改动影响。
```

例子：

- 优化 Agent 工作流入口，让日常任务默认走轻量提示词，涉及到的 SOP、导航、状态入口都同步一下。
- 检查任务卡工作流是否完整，把“已完成但仍 active”的规则写清楚并同步模板。

## 三、最好额外补一句的话

如果你愿意多写一句，最有帮助的是下面四类信息：

- **优先级**：`先做高价值部分，剩下的挂 follow-up`
- **风险偏好**：`先别批量改，先让我看报告`
- **收尾要求**：`做完同步工作日志、活跃任务和状态摘要`
- **产出偏好**：`先要备课大纲，不要直接做讲义`

## 四、最容易让 Agent 跑偏的模糊说法

| 模糊说法 | 更好的说法 |
|---|---|
| “继续做一下” | “继续推进第四轮冲刺专题，先做备课大纲，不做讲义” |
| “整理一下这个” | “把这套网课整理成教学逻辑提炼，不直接回填 KP” |
| “优化一下知识库” | “审计 03-知识点/有机化学 的 frontmatter 一致性，只修低风险问题” |
| “帮我备课” | “为 [[专题-氧化还原与电化学]] 做第一轮基础班 120 分钟备课大纲” |
| “把这个弄完整” | “把 [[某专题页]] 升级到可用，补核心结论、对比表和 2 道例题” |

## 五、复制即用模板

### 模板 A：提炼资料

```text
把【资料名】的【章节 / 范围】提炼进知识库，
只做【考纲内 / 指定主题】，
产出【资料提炼笔记 / 教学逻辑提炼】，
暂时不做【KP 回填 / 题库拆分 / 讲义】，
做完同步工作日志。
```

### 模板 B：填充 KP

```text
补全【KP 名称 / 一批 KP】，
重点做【教学视角 / 易错点 / 例题 / frontmatter】，
不要改文件名和目录结构，
做完给我一个缺口总结，并同步工作日志。
```

### 模板 C：审计问题

```text
检查【目录 / 模块】里的【问题类型】，
先只出报告 / 先修低风险问题，
不要直接做大规模重构，
最后同步工作日志和活跃任务（如果产生 follow-up）。
```

### 模板 D：备课

```text
为【专题 / KP / 课题】准备【轮次 + 班型 + 时长】的【备课大纲 / 讲义】，
重点讲【侧重点】，
暂时不做【学生侧材料 / PPT / 复盘】，
做完同步备课待办、工作日志和活跃任务。
```

### 模板 E：改工作流

```text
优化【SOP / 提示词 / 导航 / 模板】，
目标是【更省 token / 更少漏同步 / 更清楚路由】，
相关入口文件都帮我同步，
最后写工作日志，并告诉我这次改动会影响什么默认行为。
```

---

## 六、一句话原则

> **说清“做什么、做到哪、别做什么、做完同步哪里”，Agent 就会明显更稳。**
