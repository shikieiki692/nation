# 会话交接 · 说明

> **用途**：解决"下一轮 Agent 不知道本轮做了什么"的问题。
>
> 会话交接笔记是 **"增量信号"**——只记录本轮的关键事实，不替代工作日志的完整归档。
> 下一位 Agent 读交接笔记优先于读入口文件链：如果所有入口文件的 `updated` 都没变，可以直接跳过。

## Agent 启动读取规则

```
进入会话 → 检查 00-首页/会话交接/README.md
  ├── 读索引表（下方），只看最新一条
  ├── 如果入口文件 updated 没变 → 跳过重读
  ├── 如果入口文件变了 → 只读变更部分
  └── 需要历史信息 → 去 09-审计报告/会话交接归档/ 按日期范围查
```

**不读全部历史笔记**。索引表已经浓缩了每条笔记的核心产出，只用读最新那条的全量内容。

## 归档规则

| 策略 | 值 |
|:---|---:|
| 活跃区上限 | 5 条（`max_active_notes`）|
| 超出时处理 | 最老笔记移至 `09-审计报告/会话交接归档/YYYY/` |
| 归档后索引 | 索引表状态列改为"📦 归档"，保留链接 |
| 归档文件命名 | 不变（`YYYY-MM-DD-任务名.md`）|

**归档操作 SOP**（Agent 写新笔记时顺便执行）：
1. 如果当前 `会话交接/` 下 `.md` 文件（不含 README）> 5 条
2. 取按日期最老的（不含当前最新 1 条，确保正在用的不被误移）
3. 确认归档目录存在：`ls 09-审计报告/会话交接归档/YYYY/`
4. 移动：`mv 00-首页/会话交接/YYYY-MM-DD-任务名.md 09-审计报告/会话交接归档/YYYY/`
5. 更新本页索引表状态列："🟢 活跃" → "📦 归档"

## 核心规则

1. **每次 Phase 4 结束时写**：15 行内完成，存在 `00-首页/会话交接/YYYY-MM-DD-任务名.md`
2. **只记录增量**：本轮做了什么、改了哪些文件、下一轮建议
3. **不替代工作日志**：完整过程和证据仍在 [[工作日志]] 中，交接笔记只做快速恢复用
4. **Agent 启动时优先读本索引表** → 只看最新一条 → 不批量读历史

## 文件规范

```
00-首页/会话交接/

```

## 与工作日志的分工

| | 会话交接笔记 | 工作日志 |
|:---|:---|:---|
| 篇幅 | ~15 行 | 不限 |
| 读者 | 下一位 Agent（优先读索引，只读最新1条） | 人类 + Agent（回溯用） |
| 内容 | 做了什么 + 改了哪些文件 + 下轮建议 | 完整过程 + 验证 + 证据 |
| 保留策略 | 活跃目录保留 ≤5 条，超出自动归档 | 永久保留 |
