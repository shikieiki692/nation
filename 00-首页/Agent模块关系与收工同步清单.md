# Agent模块关系与收工同步清单

> **用途**：给 Agent 一份真正可在 1-2 分钟内读完的“模块地图 + 收工回写矩阵”。
> 当任务跨模块、或你不确定“该先读哪里 / 做完要回写哪里”时，优先看本页。

## 二、从用户任务反推读取路径

| 用户任务类型 | 开工前最少读取 | 产出后必须回写 |

| 专题页 / 题型页建设 | 上述三文件 + 相关 KP + 题库 + `模板-专题` | `工作日志` + `活跃任务` + 必要时 `备课与教学思路待办` |

## 四、收工同步矩阵（硬规则）

| 本次改动类型 | 必须同步 |
|---|---|
| 新增 / 修改 `07-资料提炼` | `工作日志`；若改变主线或补齐重要盲区，再同步 `状态摘要` / `Agent战略方向` |
| 新增 / 批量修改 KP | `工作日志`；必要时 `知识点总索引` / `题库链接` / `活跃任务` |
| 新增 / 升级专题页 | `工作日志`；若改变备课闭环状态，同步 `备课与教学思路待办` 和 `活跃任务` |
| 生成任何 `04-课件/` 文件 | `工作日志` + `活跃任务` + `备课与教学思路待办`（硬规则） |
| 新增非正文交付物（PPT / docx / 教具 / 工具卡 / 评分表） | `工作日志` + `活跃任务` + `04-课件/专题产出总览看板`；若改变阶段判断，再同步 `状态摘要/Agent战略方向/首页` |
| 改动 `02-数据库/数据库-真题`、题库查询口径或组卷入口 | `工作日志` + `活跃任务`；若改变默认题库使用路径，再同步 `首页/状态摘要/Agent战略方向` 中的相关描述 |
| 改动 `方法论-多源教学资料整合` 或提炼→备课闭环规则 | `工作日志` + 必要时 `Agent标准作业流程` / `Agent模块关系与收工同步清单` / `状态摘要` |
| `化学竞赛初赛讲义` 这类持续性资料线推进了默认顺序或方法 | `工作日志` + 对应任务卡；若影响入口判断，再同步 `状态摘要` / `Agent战略方向` / 本页 |
| 关闭 / 阻塞一张任务卡 | `活跃任务卡` frontmatter；必要时同步 `状态摘要` 的阻塞 / 下一步 |
| 改变阶段判断、阻塞项、下一步 | `状态摘要` + `Agent战略方向`；大任务时同步 `首页.current_focus` |
| 模板 / SOP 版本变更 | 对应模板文件 + `状态摘要` 体系版本表 + `首页.sop_version`（若改 SOP） |
| 学生侧材料试点、反馈、导出工作流 | `工作日志` + `活跃任务`；若改变学生侧主线或阻塞，再同步 `状态摘要/Agent战略方向` |

### 快速判断

- **只改过程，不改结论**：通常只写 `工作日志`
- **改了待办状态**：必须写 `活跃任务`
- **改了阶段 / 阻塞 / 下一步**：必须写 `状态摘要`
- **改了长期优先级 / 待用户决策项**：必须写 `Agent战略方向`
- **改了课件进度**：必须写 `备课与教学思路待办`

## 五、最容易漏掉的 6 件事

1. 课件生成完了，但没回写 `备课与教学思路待办`
2. 任务正文里写“已完成”，frontmatter 还是 `active`
3. 审计结论已经接受“复用已有产物”，却还挂着一个等待“补做”的旧任务
4. 任务其实在等用户 / 打印 / 课堂反馈，却一直占着 `active`
5. 改了 SOP / 模板版本，却没同步 `状态摘要` 的体系版本表
6. 学生侧材料开始试点了，但 `06-学生侧材料/`、`KP复习清单`、Anki、反馈闭环仍各写各的

## 六、推荐使用方式

### 日常小任务

```text
状态摘要 → 活跃任务 → 本页 → 开工
```

### 跨模块任务

```text
状态摘要 → 活跃任务 → Agent战略方向 → 本页 → 方法论 / SOP 对应章节 → 开工
```

### 收工前 20 秒

```text
工作日志写了吗？
任务卡状态对了吗？
如果生成了 04-课件，待办表更了吗？
如果结论变了，状态摘要 / 战略方向更了吗？
```
