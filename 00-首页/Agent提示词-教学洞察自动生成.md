# Agent 提示词：教学洞察自动生成

> 🎯 **用途**：给定一份 `教学逻辑提炼` 文件，Agent 自动生成对应的 `教学洞察` 初稿。Agent 完成 ~80% 的填充，剩余 ~20%（发生率估计、类比润色、轮次差异审核）由人工补充。
>
> 📥 **输入**：`[[教学逻辑提炼-XXX]]`
> 📤 **输出**：`12-教学洞察/教学洞察-<KP名称>.md`

---

## 执行步骤

### Step 1：读取源文件

读取目标 `教学逻辑提炼` 文件全文，确认以下板块存在且内容充足：

| 必需板块 | 用途 | 若缺失 |
|:---|:---|:---|
| §四 认知台阶总表 | → 洞察 §一 认知台阶链 | ⚠️ 标记「台阶信息不足」 |
| §四 教学切入点 | → 洞察 §二 关键认知冲突 | ⚠️ 仅从 §八 提取 |
| §六 例题/活动总串 | → 洞察 §三 例题选用指南 | ⚠️ 标记「缺少例题指南」 |
| §八 关键教学语言段 | → 洞察 §二/§四/§五 | ⚠️ 若缺失则洞察质量降级，无法自动生成类比/口诀 |
| §九 轮次深度边界建议 | → 洞察 §六 轮次差异 | ⚠️ 标记「缺少轮次信息」 |

### Step 2：逐板块映射填充

按以下映射表，从教学逻辑提炼中提取内容，填入教学洞察模板。

#### 2.1 洞察 §一「认知台阶链」← 教学逻辑提炼 §四/§五

```markdown
## 一、认知台阶链

> 学生从「现状」到「掌握」，需要跨过几级台阶？每级的关键跨越是什么？

| 台阶 | 学生现状 | 目标状态 | 关键跨越 |
|:---:|:---|:---|:---|
| 1 | <取自 §五「学生已有认知」列> | <取自 §五「新认知」列> | <取自 §五「讲解策略」列，提炼为一句话> |
```

**自动填充规则**：
- 只选取 `深度标记 = 第一轮必讲` 的行
- 「关键跨越」从「讲解策略」列中提炼为核心动作（如「反例驱动：先用错误 R 值算→再纠正」→ 提炼为「经历一次『算错了』的冲击」）
- 台阶数建议 3-7 个

**需人工判断**：台阶顺序是否需要调整（教学逻辑提炼中的顺序 = 原资料顺序，洞察中可以按认知难度重排）。

#### 2.2 洞察 §二「关键认知冲突」← 教学逻辑提炼 §四 + §八 §8.1

```markdown
## 二、关键认知冲突

### 冲突 N：<冲突名称>（发生率 ~XX%）
- **制造方式**：<取自 §八 §8.1 对应冲突的「教师原话」，改写为教学操作描述>
- **突破策略**：<取自 §八 §8.1 + §四「教师如何铺垫」>
```

**自动填充规则**：
- 从 §八 §8.1 中提取每条「制造认知冲突的原话」，一个冲突一个子节
- 「制造方式」从教师原话改写为操作描述（"老师说：来大家用 R=8.31 算一下…" → "给同一组数据，让学生分别用不同 R 值计算，对比结果的百倍差异"）
- 「发生率」默认填 `~XX%`，等待人工补充

**需人工补充**：发生率估计（基于教学经验或作业反馈数据）。

#### 2.3 洞察 §三「例题选用指南」← 教学逻辑提炼 §六

```markdown
## 三、例题选用指南

> 例题的功能 > 例题的数量。每道题标注教学功能和使用时机。

| 序号 | 例题 | 教学功能 | 何时用 | 来源 |
|:---:|:---|:---|:---|:---|
| 1 | <取自 §六「例题/活动」列> | <取自 §六「作用」列> | <取自 §六「出现位置」列 + 推断> | <取自 §六 来源或 §一> |
```

**自动填充规则**：
- 全量导入 §六 表格
- 「何时用」列优先使用 §六「出现位置」（概念前/概念后/总结后），补充具体时机（如「讲 R 值之前」「讲完密度形式后」）
- 标注 `当前轮次处理` 不为「直接使用」的例题为 `> ⚠️ 后续轮次`

**需人工判断**：例题顺序是否合理（洞察中应按教学功能排序：冲突制造 → 建立肌肉记忆 → 变形应用 → 综合）。

#### 2.4 洞察 §四「高频错误模式」← 教学逻辑提炼 §八 §8.3 + 教学洞察已有内容

```markdown
## 四、高频错误模式

| 错误 | 典型表现 | 发生率估计 | 预防策略 |
|:---|:---|:---:|:---|
| <错误名称> | <取自 §8.3「预防错误的警告语」中的错误描述> | ~XX% | <取自 §8.3 警告语中的纠正动作> |
```

**自动填充规则**：
- 从 §八 §8.3 逐条提取，一条警告 = 一个错误行
- 「发生率」默认 `~XX%`，等待人工补充
- 如果教学逻辑提炼中没有 §八，尝试从 §四「教学切入点」的「第一处认知冲突」和 §五 的「讲解策略」列中提取分散的错误描述

**优先级规则**：如果目标 KP 已有教学洞察文件（含错误模式），**保留已有发生率数据**，仅补充 §八 §8.3 中新增的错误条目。

#### 2.5 洞察 §五「类比与口诀」← 教学逻辑提炼 §八 §8.2 + §8.3/§8.4

```markdown
## 五、有效类比与口诀

### 类比库
| 概念 | 类比 | 使用场景 |
|:---|:---|:---|
| <取自 §8.2「概念」列> | <取自 §8.2「比喻」列> | <取自 §8.2「教师近原话」中推断的使用时机> |

### 口诀库
| 口诀 | 用途 |
|:---|:---|
| <从 §8.3 和 §8.4 中提取的短句/顺口溜> | <用途描述> |
```

**自动填充规则**：
- 类比：全量导入 §8.2 表格，一行为一条类比
- 口诀：从 §8.3「警告语」和 §8.4「过渡语」中提取可压缩为口诀的短句（如「恒容求分压，恒压求分体积」← 来自「条件没读就开始套公式」的反面表述）
- 如果教学逻辑提炼中没有 §八，从已有教学洞察文件迁移（若存在）

**需人工补充**：润色口诀为押韵/对仗形式，补充教学洞察已有但 §八 未收录的类比。

#### 2.6 洞察 §六「轮次差异」← 教学逻辑提炼 §九 + §十

```markdown
## 六、轮次差异

| 维度 | 第一轮 | 第二轮 | 第三轮 |
|:---|:---|:---|:---|
| 深度边界 | <取自 §九「本轮讲到哪里为止」> | <取自 §九 第二轮行> | <取自 §九 第三轮行> |
| 禁止提前展开 | <取自 §十> | — | — |
| 典型例题难度 | ⭐⭐-⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
```

**自动填充规则**：
- 全量导入 §九 和 §十 表格
- 若 §九 或 §十 缺失，仅保留第一轮列，其余标记 `⚠️ 待补充`

#### 2.7 洞察 §七「来源追溯」← 全文件

```markdown
## 七、来源追溯

| 洞察 | 主要来源 | 置信度 |
|:---|:---|:---:|
| 认知台阶链 | [[教学逻辑提炼-XXX]] §五 | 高 |
| 关键认知冲突 | [[教学逻辑提炼-XXX]] §四 + §八 §8.1 | 高 |
| 例题选用指南 | [[教学逻辑提炼-XXX]] §六 | 高 |
| 高频错误模式 | [[教学逻辑提炼-XXX]] §八 §8.3 | 中（发生率需实测验证） |
| 类比与口诀 | [[教学逻辑提炼-XXX]] §八 §8.2 | 中（近原话重构，非逐字稿） |
| 轮次差异 | [[教学逻辑提炼-XXX]] §九 + §十 | 高 |
```

**置信度评级规则**：
- 来自教学逻辑提炼结构化表格（§四/§五/§六/§九/§十）→ **高**
- 来自 §八 关键教学语言段（近原话重构）→ **中**（标注「近原话重构，非逐字稿」）
- 发生率估计 → **中**（标注「发生率需实测验证」）

### Step 3：写入文件

生成完整的教学洞察 Markdown 文件，写入 `12-教学洞察/教学洞察-<KP名称>.md`。

若文件已存在（轻量版），追加缺失板块而非覆盖已有内容。合并规则：
- 已有板块 → 保留，仅补充教学逻辑提炼中新增的条目
- 缺失板块 → 按 Step 2 自动填充
- 发生率数据 → 保留已有（已有人工审核），新增条目默认 ~XX%

### Step 4：输出摘要

生成完成后，输出摘要：

```
📊 教学洞察生成摘要

文件：[[教学洞察-XXX]]
来源：[[教学逻辑提炼-XXX]]

自动填充完成：
✅ §一 认知台阶链 —— <N> 级台阶
✅ §二 关键认知冲突 —— <N> 个冲突
✅ §三 例题选用指南 —— <N> 道例题
✅ §四 高频错误模式 —— <N> 条错误
✅ §五 类比与口诀 —— <N> 条类比 + <N> 条口诀
✅ §六 轮次差异 —— <N> 轮
✅ §七 来源追溯 —— 完成

待人工补充：
⚠️ 发生率估计（§二 各冲突 + §四 各错误）
⚠️ 口诀润色（§五）
⚠️ 第二轮/第三轮轮次差异细化（§六）
```

## 注意事项

1. **不编造内容**：所有自动填充内容必须能在教学逻辑提炼中找到对应来源。无法填充的板块标记 `⚠️ 待从教学逻辑提炼补充`，不要用通用知识填充。
2. **发生率保守**：发生率一律默认 `~XX%`，不编造数字。若已有教学洞察文件含发生率，保留已有数据。
3. **合并优先**：若目标洞察文件已存在（即使是轻量版），合并而非覆盖。已有的人工审核内容（发生率、润色后的口诀）优先保留。
4. **标注来源**：每条洞察都标注来自教学逻辑提炼的哪个章节，方便人工追溯验证。
