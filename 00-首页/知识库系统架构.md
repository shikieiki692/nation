# 知识库系统架构

> **用途**：说明知识库各系统层、模块间关系、数据流向与职责边界。
> 若只想快速理解"当前该做什么"，走 [[状态摘要]] → [[活跃任务]] → [[Agent战略方向]]。
> 若想理解首页各入口文件的关系，见 [[Agent入口文件关系说明]]。

## 一、系统层架构

知识库按"资料流入 → 提炼吸收 → 知识组织 → 教学产出"的纵向链路分层：

```
┌─────────────────────────────────────────────────────────────┐
│  🎯 考纲层 (01-考纲导航/ + 02-考纲条目/)                    │
│  定义边界：什么该覆盖、什么不该覆盖                          │
│  不承载正文主内容，仅提供条目映射与覆盖审计                   │
└──────────────────────────┬──────────────────────────────────┘
                           ↓ 条目映射
┌──────────────────────────┴──────────────────────────────────┐
│  📥 原始资料层 (06-外部资料导入/)                           │
│  存放教材扫描件 / 网课笔记 / 真题 PDF 等原始文件             │
│  状态：OCR待整理 / 已导入待提炼                              │
└──────────────────────────┬──────────────────────────────────┘
                           ↓ 提炼
┌──────────────────────────┴──────────────────────────────────┐
│  🔄 提炼层 (07-资料提炼/)                                   │
│  将外部资料转为可消费资产：                                 │
│  ├─ 章节/书籍提炼：知识、例题、图表、考纲映射              │
│  └─ 教学逻辑提炼：认知台阶、轮次边界、可回流资产           │
│  提炼成果的下游消费者：KP、专题页、备课大纲                 │
└──────┬──────────────────────────────────┬───────────────────┘
       ↓ 知识吸收                         ↓ 教学逻辑吸收
┌──────┴──────────────────┐  ┌──────────┴────────────────────┐
│  📚 知识组织层          │  │  🧩 训练组织层                │
│  (03-知识点/)            │  │  (04-专题与题型/ + 04-题库/) │
│  KP：定义/原理/易错点    │  │  专题页：知识模块+解题引擎   │
│  教学视角/例题/对比表    │  │  题型页：方法聚合             │
│  当前：教学层已全线收束  │  │  题库：真题+习题结构化        │
└──────┬──────────────────┘  └──────┬────────────────────────┘
       │ ↔ 双向回流                 │ ↔ 双向链接
       ↓                             ↓
┌─────────────────────────────────────────────────────────────┐
│  🎓 教学产出层 (04-课件/)                                   │
│  ├─ 备课大纲：教师教学设计（认知台阶/课时/媒介）            │
│  ├─ 学生讲义：课堂材料（知识/例题/易错点/作业）            │
│  ├─ 习题课/复习课：真题重组                                 │
│  └─ 课堂执行页：投影速查卡+追问链                           │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌──────────────────────────┴──────────────────────────────────┐
│  📝 学生侧材料 (06-学生侧材料/)                             │
│  闪卡 / 练习卷 / 策略卡 / Anki 导出                        │
│  状态：模板就绪，待试点量产                                 │
└─────────────────────────────────────────────────────────────┘
```

## 二、模块关系总览

### 核心链路

```
教材/网课/真题
    ↓ 导入
06-外部资料导入
    ↓ 提炼
07-资料提炼
    ├──→ 03-知识点（source_notes / source_extracts）
    ├──→ 04-专题与题型（专题页素材）
    └──→ 04-课件（备课大纲的教学逻辑依据）
03-知识点 ↔ 04-题库（knowledge_points 字段双向链接）
03-知识点 + 04-专题与题型 + 07-教学逻辑提炼
    ↓
04-课件/备课大纲（教师设计）
    ↓
04-课件/学生讲义（课堂材料）
    ↓
06-学生侧材料（课后复用）
```

### 系统控制层

```
00-首页/ 系统入口层
├── 状态摘要      — 会话快照（Agent 首读）
├── 活跃任务      — 短期执行聚合 + 任务卡健康检查
├── Agent战略方向  — 长期优先级 + 默认推进路径
├── 首页           — 人工操作台（备课/建设/导航）
├── Agent标准作业流程 — 完整工作流规则书
├── 经验沉淀       — 踩坑速查全集
└── 图片索引       — 图片资产仓库入口
```

## 三、数据流向规则

| 步骤 | 操作 | 数据去向 | 关键约束 |
|:---:|:---|:---|:---|
| ① | 导入外部资料 | `06-外部资料导入/` | 原始文件，不做内容修改 |
| ② | 提炼（每章/总索引/网课思路） | `07-资料提炼/` | 必须补"备课抽料资产标记"才算接入工作流 |
| ③ | 回流到 KP | `03-知识点/`（`source_notes`/`source_extracts`/教学视角） | 同一知识点只落一个 KP，用来源字段区分 |
| ④ | 回流到专题页 | `04-专题与题型/` | 提炼成果→专题页的"相关提炼"链接 |
| ⑤ | 备课 | `04-课件/备课大纲/` | 必须先确认教学逻辑边界和轮次深度 |
| ⑥ | 产出讲义 | `04-课件/学生讲义/` | 基础班结论后置；例题 step-by-step |
| ⑦ | 课后复用 | `06-学生侧材料/` | 闪卡/练习卷/策略卡按模板产出 |
| ⑧ | 收工记录 | `00-首页/工作日志/` + 活跃任务卡 | 必须写日志、更新任务卡状态 |

## 四、模块职责速查

| 目录 | 核心职责 | 不做什么 |
|:---|:---|:---|
| `01-考纲导航/` `02-考纲条目/` | 考纲边界定义与条目级映射 | 不承载正文内容 |
| `03-知识点/` | 定义、原理、易错点、教学视角 | 不做解题方法聚合（那是专题页的事） |
| `04-专题与题型/` | 知识模块+解题引擎 | 不做逐课教学设计（那是备课大纲的事） |
| `04-题库/` | 题目结构化存储与筛选 | 不做组卷后的教学编排 |
| `04-课件/` | 备课大纲 + 讲义 + 习题课 | 不做知识点级的定义（那是 KP 的事） |
| `06-外部资料导入/` | 原始资料暂存 | 不做提炼、不做内容修改 |
| `07-资料提炼/` | 外部资料→可消费资产 | 不做备课级教学设计 |
| `06-学生侧材料/` | 闪卡/练习卷/策略卡产出 | 不做课堂教学设计 |
| `09-审计报告/` | 审计/断链/归档 | 不做日常执行判断 |
| `11-模板/` | 模板 + Agent 提示词 | 不承载教学内容 |
| `00-首页/` | 系统入口 & 治理 | 不承载学科内容 |

## 五、关键连接说明

### 资料提炼 → KP 的连接

提炼页产出的核心内容应通过以下字段回流到 KP：
- **`source_notes`**：引用来源资料/提炼页
- **`source_extracts`**：直接引用的原文或提炼结论
- **§12 教学视角**：填入从提炼中提取的认知难点、教学建议

### KP → 题库的连接

题库题目的 `knowledge_points` 字段应链接到对应的 KP：
- 一道题可链接多个 KP
- 一个 KP 应至少链接 1 个典型题目
- 真题优先使用 `[[真题-届数-题号]]` 命名规范

### 专题页 → 备课的连接

专题页是备课的素材中枢：
- 备课大纲的 §一 总览表应包含"对应专题页"wikilink 列
- 专题页底部的 `📎 相关提炼` 应链接到对应的资料提炼页
- 专题页的真题推荐表应使用 Dataview 从题库自动查询

## 六、跨系统维护原则

1. **真相源唯一**：每个事实只在一个位置维护（如任务状态只在任务卡 frontmatter，不在工作日志正文中手写）
2. **自动汇总优于手写**：可计算的数据（库存、版本、任务状态）优先用 Dataview 自动聚合，不手写静态数字
3. **归档不删除**：历史文件移入 `09-审计报告/` 相应子目录，不直接删除（保留 wikilink 兼容性）
4. **入口三层分离**：
   - 会话快照 ← `状态摘要`
   - 短期执行 ← `活跃任务` + 任务卡
   - 长期方向 ← `Agent战略方向`
5. **基建不做重复建设**：当前已从"补充有没有"转入"精品化与维护期"，新模块建设前先检查现有资产是否已覆盖
