# Agent 任务分解指南

> 当用户的一句话跨多个 Agent 时，如何拆分、排序、并行。
> 本指南由 SOP Phase 1 调用，帮助 Plan Agent 快速生成执行计划。
>
> **何时读**：任务需拆成 ≥3 个子任务时。日常单步骤任务走 [[Agent最小执行协议#⚡ Agent 文件速查表]]。<br>
> **读完需时**：~1.5 min。

> **默认口径（2026-06-07 起）**：拆分前先按 [[Agent最小执行协议]] 判断是否只需“轻量提示词 + 模板”。只有复合链路、批量任务、迁移/审计类任务，才需要把本指南完整用起来。

> **页面边界（2026-06-15）**：本页只回答“任务怎么拆”。它不是日常默认入口，也不负责判断当前阶段重点、数据库怎么用、收工要回写哪里。
> - 想知道现在先做什么：去 [[状态摘要]]、[[活跃任务]]、[[Agent战略方向]]
> - 想知道上下游与收工同步：去 [[Agent模块关系与收工同步清单]]
> - 想知道 `.base` / 工作日志 / 任务卡怎么配合：去 [[Agent数据库使用指南]]

## 推荐入口（先用这里）

```text
普通任务
  -> [[Agent最小执行协议]]

跨两个及以上模块，但链路仍清楚
  -> [[Agent最小执行协议]] + [[Agent模块关系与收工同步清单]]

需要真正拆成多子任务 / 多批次 / 并行执行
  -> 本页
```

> **⚡ 子 Agent 调度规则（与 [[Agent最小执行协议#一、任务驱动执行流程（默认模式）]] 配合）**：
> 1. **每个子 Agent 必须有独立任务卡**：主 Agent 拆分时为每个子任务建卡，填写 `task_type`
> 2. **`task_type` 决定子 Agent 路由**：子 Agent 不读全套入口文件，按路由表确定读什么、用什么提示词、收工同步哪里
> 3. **子 Agent 收工先更新自己的任务卡**：`status=completed` + `evidence` 必填
> 4. **串行链依赖上游任务卡状态**：下游启动前先确认上游任务卡 `status=completed`
> 5. **每个子 Agent 独立写会话交接笔记**：存 `00-首页/会话交接/YYYY-MM-DD-子任务名.md`
> 6. **主 Agent Phase 4 汇总全部子任务卡 evidence**：确认全部 completed 后再同步入口页

## 0.5 先决定用轻量还是长版

| 场景 | 读取策略 |
|---|---|
| 单一 Agent、单模块、边界清楚 | 直接用对应 `Agent-*-轻量` |
| 多 Agent 串行但规则普通 | 轻量提示词 + 本指南拆分 |
| 迁移 / 合并 / 弃用 / 全库审计 | 本指南 + 长版提示词 + 完整 SOP |
| 改模板 / 改 SOP / 改系统入口 | 完整 SOP 必读 |

## 常见复合任务拆分模板

### 模板 A：教材 → 知识点 → 题目（完整管线）
**用户原话示例**："帮我把《普通化学原理》第3章的内容整理进知识库，再把习题拆成单题"

**拆分**：
```
Phase 1: 确认源文件存在 + 确认考纲范围
  ↓
Agent 1（资料提炼）→ 07-资料提炼/普化原理-第3章.md
  ↓ 串行依赖
Agent 2（知识点填充）→ 03-知识点/化学原理/...（新知识点或填充已有骨架）
  ↓ 串行依赖
Agent 4（题库拆分）→ 04-题库/化学原理/...（从提炼笔记的例题/习题中提取）
  ↓ 串行依赖
Agent 5（交叉链接）→ 补全知识点↔题目双向链接
```

**批次策略**：
- Agent 1：1个Agent处理1章
- Agent 2：若知识点 >10个，分2-3批，每批5-8个
- Agent 4：若题目 >20道，分2批
- Agent 5：1个Agent处理1个科目模块

**预估产出**：1份提炼笔记 + N个知识点 + M道单题

---

### 模板 B：为多个专题/题型补充例题
**用户原话示例**："给5个专题页各补3道典型例题"

**拆分**：
```
Phase 1: 确认5个专题页存在 + 从题库中检索关联题目
  ↓
并行启动 5 个 Agent 10（备课）子任务：
  - 子任务1: 专题-SN1与SN2比较 → 选3题
  - 子任务2: 专题-取代与消除竞争 → 选3题
  - ...
```

**批次策略**：
- 5个专题完全独立，可**全并行**
- 每个子任务 Agent 的 prompt 必须自包含：专题路径 + 关联知识点列表 + 题库筛选条件

**预估产出**：5个专题页各增加3道例题（含分析+解答+反思）

---

### 模板 C：全库健康检查
**用户原话示例**："扫一遍知识库，看看有什么问题"

**拆分**：
```
Phase 1: 明确检查维度（默认全量）
  ↓
Agent 9（健康检查）分批扫描：
  - 批次1: 03-知识点/化学原理（~50个）
  - 批次2: 03-知识点/无机和结构化学（~65个）
  - 批次3: 03-知识点/有机化学（~313个，可分4-5批）
  - 批次4: 03-知识点/分析化学 + 决赛要求（~80个）
  - 批次5: 02-考纲条目（~72个）+ 04-题库抽样
```

**批次策略**：
- 按科目模块分5批并行
- 每批产出一份健康检查报告
- Phase 3 汇总5份报告，去重后生成总报告

**预估产出**：1份汇总健康检查报告 → 08-待审核区/或 09-审计报告/

---

### 模板 D：新建一批知识点 + 关联考纲
**用户原话示例**："创建20个缺失的知识点，并把它们挂到对应考纲条目下"

**拆分**：
```
Phase 1: 确认20个知识点名单 + 对应考纲条目 + 是否有提炼笔记来源
  ↓
Agent 2（知识点填充）分批创建20个知识点
  - 批次1: 基础-无机3个 + 基础-分析1个（4个）
  - 批次2: 基础-有机10个（分两批，每批5个）
  - 批次3: 决赛要求6个（分两批，每批3个）
  ↓ 串行依赖
Agent 5（交叉链接）→ 在考纲条目中添加新建知识点的wikilink
  ↓ 串行依赖
Agent 9（健康检查抽样）→ 抽查5个新建知识点的格式合规性
```

**批次策略**：
- 新建知识点可以按科目并行（4批）
- 链接补全和审查必须在新建完成后串行执行

---

### 模板 E：优化一批已有知识点
**用户原话示例**："把化学原理的知识点再完善一下"

**拆分**：
```
Phase 1: 明确优化维度（深度/例题/教学视角/链接完整性）
  ↓
Agent 7（KP优化）分批处理
  - 若优化维度统一（如"全部补教学视角段"）：1个Agent处理1个科目
  - 若优化维度多样：每批只处理1个维度，避免Agent混淆
  ↓
Agent 6（质量审查）→ 对优化后的知识点做抽样审查
```

**批次策略**：
- 化学原理~50个可分2批（每批~25个）
- 每批的prompt必须明确优化维度（只补教学视角 / 只补例题 / 全面优化）

---

### 模板 F：合并重复知识点
**用户原话示例**："这俩知识点是不是同一个，能合并吗"

**拆分**：
```
Phase 1: 确认待合并的两个KP + 读取双方全文 + 列出所有引用它们的文件
  ↓
Agent 8（KP合并/弃用）执行7天软弃用流程
  - 步骤1: 在待弃用KP上标记 `status: 待弃用`，添加重定向注释
  - 步骤2: 更新所有引用该KP的文件（wikilink改为指向保留的KP）
  - 步骤3: 在 `[[活跃任务]]` 登记待跟踪项，并在工作日志记录弃用提案和7天到期日
  ↓
7天后（或用户提前确认）
  - 正式删除/归档待弃用KP
  - 更新索引
```

**注意**：合并操作是 P2 级，必须用户确认后才能执行。

## 并行 vs 串行决策树

```
任务之间是否有文件范围重叠？
  ├─ 是 → 必须串行（避免脏数据）
  └─ 否 → 是否有逻辑依赖（B需要A的产出）？
        ├─ 是 → 串行
        └─ 否 → 可并行

特例（即使无重叠也建议串行）：
- 操作同一 frontmatter 字段的批量修改（避免格式漂移）
- 新建目录结构后立即向其中写文件（目录创建有延迟风险）
```

## 子任务 prompt 必备要素

每个并行子 Agent 的 prompt 必须包含：

1. **文件范围**（精确路径或Glob模式）
2. **模板引用**（[[模板-XXX]]）
3. **DoD（完成定义）**（3-5条检查项）
4. **边界条件**（明确不做什么）
5. **输出格式**（frontmatter必填字段、正文结构）
6. **异常处理**（遇到缺失/错误时怎么办）
7. **任务卡路径**（指定本子任务对应的任务卡文件，子 Agent 开工前先读卡确认状态）
8. **会话交接要求**（收工前必须写 `00-首页/会话交接/YYYY-MM-DD-子任务名.md`，15 行内描述：做了什么、改了哪些文件、下轮建议）
