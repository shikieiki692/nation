当前管线图片策略

1. 源文件层（04-课件/学生讲义/*.md）

- 讲义图片引用统一使用规范写法 `![[media/文件名.ext]]`
- 图片引用格式：![[media/文件名.ext]] 或 ![[media/文件名.ext|宽度]]
- 文件名可以是中文、UTF-8 字符、可以是 .jpg / .png / .svg / .webp

2. 图片同步（vault → 缓存）

convert_handout_to_pdf.py 第0步执行：

vault media/  ──copy──►  11-模板/scripts/.chem_media/  (workspace-local 编译缓存)

两轮复制：
1. 全量复制：media/ 下所有文件 → 11-模板/scripts/.chem_media/（保留原名）
2. 别名复制：chem_media_aliases.json 中的映射，把中文名文件以英文名再复制一份过去（供 LaTeX 用 ASCII 引用）

之后：
- SVG→PNG 自动转换：对 11-模板/scripts/.chem_media/ 下的 .svg 尝试用 cairosvg 渲染同名的 .png（失败则跳过）
- 魔数校验：检查每个 .jpg/.png 的文件头（JPG 必须是 FFD8，PNG 必须是 89504E47），如果发现是 SVG 内容写了假扩展名 → 删除伪文件

3. 预处理引用转换

preprint() 中：

![[media/xxx.ext]]    →    ![](media/xxx.ext)
![[media/xxx.md]]     →    ![](media/xxx.png)       （Excalidraw .md源→渲染图）

图片路径保留 media/ 前缀，不改文件名。

4. Pandoc + Lua filter

- Pandoc 把 Markdown 转 LaTeX，图片变成 \includegraphics{media/xxx.png}
- wrap_images.lua 检测纯图片段落 → 包裹成 \begin{figure}[H]...\end{figure}
- 图片后紧跟着的 *图 xxx*（Emphasis 节点）→ 自动变成 `\caption{...}`，并自动剥掉手写的“图 N”前缀

5. LaTeX 后处理

- {media/ 前缀被剥掉（因为 \graphicspath 已经指向 11-模板/scripts/.chem_media/，路径多余）
- 别名映射：\includegraphics{中文名.png} → \includegraphics{ascii_name.png}
- \pandocbounded{ 剥离 + 修复多余的 }
- caption 中 _ 转义（避免数学模式断裂）

6. 缺图检查

管线扫描所有 \includegraphics，检查 11-模板/scripts/.chem_media/ 下是否有对应文件。没有则替换为 \textbf{[⛔ 缺图: xxx]}，并尝试 .jpg ↔ .png ↔ .gif 扩展名变体补救。

---
一句话总结：正式图片单一路径维护在 `media/`，管线只在缓存层做同步、SVG转PNG、别名映射和魔数校验，源文件不动一分。
