# 样板专题正式产物验收标准

> 用途：统一判断一个“样板专题”是否已经从“课堂可用”推进到“正式产物稳定”。
>
> 适用对象：
> - `04-课件/加工页/*-课件加工页`
> - `04-课件/专题产出总览看板`
> - `备课思路/审计归档/精品化剩余工作清单`

---

## 一、三层完成定义

## 零、前置门槛

> 从 2026-06-19 起，`PPT` 不再作为样板专题收口的起点，而是默认作为最后一跳。

### 0.1 先后顺序硬门槛

1. 先冻结 `备课大纲`
2. 再冻结 `学生讲义 / 新授课正文`
3. 最后才进入 `PPT`

### 0.2 未满足以下条件时，不应开始做 PPT

- 备课大纲中的课时结构、例题顺序、板书/PPT 分工仍在频繁改动
- 学生讲义的主链、机制解释、例题链、易错点仍未讲透
- 大纲和讲义对“本节到底讲到哪里”仍有口径偏差
- 当前只是“有一个能投的骨架”，但尚未确认学生侧材料是否真可发放

### 0.3 开始做 PPT 的最低准入条件

- 教师上课大纲已经能支持教师不回头查正文也能上课
- 学生讲义已经完成至少一轮“五件套”核对
- 大纲与讲义的主问题、判断顺序、例题顺序一致
- 已明确哪些内容必须投影，哪些内容应留在板书或讲义

### 1.1 课堂可用

满足以下条件即可判为“课堂可用”：

- 备课大纲已成型
- 新授课 / 学生讲义主正文已成型
- 主线、例题、边界、作业已具备基本闭环

### 1.2 课堂精品

在“课堂可用”基础上，继续满足：

- 已补 `课堂投影速查卡`
- 至少补了 1 组 `第二张教学图` 或 `讲评映射板`
- 已形成固定课堂口令、板书模板或题型统一问法
- 图示、板书、讲评之间已有明确媒介分工

### 1.3 正式产物稳定

在“课堂精品”基础上，继续满足：

- 教师上课大纲 `.docx` 已导出并验收
- 学生讲义 `.docx` 已导出并验收
- 投影课件 `.pptx` 已完成终版并通过“投影优先”检查
- 看板与加工页已同步回写，不再存在“正文已好但看板仍写进行中”的状态错位

## 二、PPT 终版验收标准

### 2.1 结构层

- [ ] 已形成清晰页序，而不是零散素材拼接
- [ ] 开场页能明确本节主问题
- [ ] 中段页按 `概念/图景/规则/例题/收束` 分工承载
- [ ] 结尾页能回到判断顺序或总结构架

### 2.2 投影层

- [ ] 单页只承载一个核心任务
- [ ] 长文本已压缩成短句、图、表或流程
- [ ] 例题页只投题干与提示，不全文投答案
- [ ] 必须投影的关键图已经进入 PPT
- [ ] 不适合投影的长推理已明确留给板书或讲义

### 2.3 课堂执行层

- [ ] PPT 与板书的分工已写清
- [ ] 至少有 1 页可作为课尾“判断顺序卡”或“讲评总框架”
- [ ] 学生看完 PPT 能复述本节主线，而不是只记住零散结论

### 2.4 技术层

- [ ] 字号、行距、图表密度适合投影
- [ ] 上下标、公式、离子符号显示正常
- [ ] 图片清晰度足够，未出现整页教材截图硬塞
- [ ] 至少完成 1 轮 inspect / render / 人工预览检查

---

## 三、docx 验收标准

### 3.0 大纲 / 讲义优先原则

- [ ] 本专题不是先做 PPT 再倒逼大纲和讲义补课
- [ ] 大纲与讲义已先于 PPT 达到稳定可教、稳定可发放状态
- [ ] 若 PPT 发现内容过少或逻辑不顺，优先回查大纲与讲义，而不是先堆动画或改排版

### 3.1 教师上课大纲

- [ ] 课时结构、边界、例题顺序、课堂口令齐全
- [ ] 能直接支持教师上课，不依赖回头查正文
- [ ] 已完成 1 轮导出检查

### 3.2 学生讲义

- [ ] 核心板块已过“五件套”检查
- [ ] 教师侧边界语句未泄漏到学生稿
- [ ] 图片、表格、上下标、分页已完成 1 轮检查
- [ ] 可打印、可发放，不是网页草稿排版

---

## 四、回写标准

一个专题只有在以下三处都同步后，才可判为“正式产物稳定”：

1. 对应加工页已写清：
   - 已完成什么
   - 还剩什么
   - 当前缺口是否只剩 PPT 或反馈链
2. `04-课件/专题产出总览看板` 已同步真实状态
3. `备课思路/审计归档/精品化剩余工作清单` 已同步该专题是否仍属“半完成待补”

---

## 四点五、非正文验收层收口标准

当系统进入“正文已收口、非正文待落地”阶段时，还应额外满足：

- [ ] 已有一张总表明确区分“正文已完成”和“非正文剩余工作”
- [ ] 已有一张执行清单明确非正文工作包顺序
- [ ] 活跃任务中已有明确主执行卡，而不是继续复用旧正文任务
- [ ] 首页 / 状态摘要 / 最小协议 / 收工同步矩阵已同步同一口径
- [ ] 至少跑通 1 次“可打印或可预览”的真实试印链，避免纸质排版长期停留在口头 blocked

## 五、当前两门样板专题的直接使用口径

### 5.1 原子结构基础（基础班）

- 当前判断：`docx 稳定，PPT 待终版`
- 下一步：按加工页中的 `投影课件终版最小模板` 直接做 PPT 收口

### 5.2 晶体结构类型与稳定性（基础班）

- 当前判断：`主课页与加工页已收口，PPT 待终版`
- 下一步：按“三组必须投影图 + 两套统一问法”完成样板级 PPT

## 六、维护规则

1. 本页是“统一尺子”，不直接代替各专题加工页。
2. 各专题若进入正式产物收口阶段，必须至少引用一次本页口径。
3. 若后续形成更成熟的 `PPT inspect / render` 工作流，本页再补技术细则，不在当前阶段提前做重。
4. 若后续出现“PPT 明显比大纲/讲义先成型”的情况，应视为流程倒置，优先修正流程，不优先修饰 PPT。
